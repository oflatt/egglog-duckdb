1//! # egglog-backend-trait
2//!
3//! A backend-agnostic interface to an egglog egraph. This crate exposes the
4//! [`Backend`] trait, along with companion traits ([`RuleBuilderOps`],
5//! [`BaseValuePool`], [`ContainerPool`]), so that the frontend `EGraph` in
6//! the top-level `egglog` crate can drive either the in-memory reference
7//! backend (`egglog_bridge::EGraph`) or the DuckDB-backed backend
8//! (`egglog_bridge_duckdb::EGraph`) through a single dyn-compatible API.
9//!
10//! ## Design principles
11//!
12//! - **Minimal IR changes.** This crate intentionally does NOT introduce a new
13//!   neutral rule IR. [`RuleBuilderOps`] mirrors `egglog_bridge::RuleBuilder`
14//!   one-for-one. The reference backend's `RuleBuilderOps` impl is a trivial
15//!   passthrough; the DuckDB backend's impl accumulates calls into its
16//!   internal data IR and submits them to the existing `compile_rule`
17//!   pipeline on `build()`. Frontend code (`BackendRule` in
18//!   `src/lib.rs::EGraph`) is unchanged in shape.
19//! - **Basic id and config types live here.** As of Phase 2 Commit 3,
20//!   `FunctionId`, `RuleId`, `ColumnTy`, `FunctionRow`, `FunctionConfig`,
21//!   `MergeFn`, `DefaultVal`, `QueryEntry`, `Variable`, and `VariableId` are
22//!   defined in this crate. `egglog-bridge` re-exports them so existing
23//!   callers continue to work. `Value`, `BaseValueId`, `ContainerValueId`,
24//!   `ExecutionState`, `ExternalFunction`, and `ExternalFunctionId` remain in
25//!   `egglog-core-relations` (already a neutral crate) and are re-exported
26//!   here for caller convenience.
27//! - **`Backend` is `Send + Sync` and dyn-compatible.** Methods that need
28//!   `T: BaseValue` or `C: ContainerValue` are factored onto
29//!   [`BaseValuePool`] and [`ContainerPool`], which expose Any-based dynamic
30//!   dispatch. A small set of generic helper functions (see the bottom of
31//!   this module) reintroduce the per-`T` sugar on top of those dyn traits.
32//! - **Cloning.** Backends must be cloneable via [`Backend::clone_boxed`].
33//!   The reference backend already derives `Clone`; DuckDB will need a
34//!   bespoke `clone_boxed` (e.g. database snapshot or replay buffer) when
35//!   it implements this trait.
36//!
37//! ## What is intentionally NOT in this trait
38//!
39//! - `with_execution_state`. The four callers in `src/` are migrated to
40//!   dedicated trait methods in a follow-up commit; this keeps `Backend`
41//!   dyn-compatible and avoids leaking the lifetime semantics of
42//!   `ExecutionState`.
43//! - `TableAction` / `UnionAction`. Under the minimal-change posture these
44//!   remain inherent methods on the bridge. The frontend keeps using them
45//!   directly. Backends that don't support them (DuckDB) error at the
46//!   primitive-registration call sites that touch them. The
47//!   [`Backend::supports_inline_table_lookups`] capability flag gates this.
48//! - A neutral rule IR. [`RuleBuilderOps`] is the seam instead.
49

98/// Properties of a function added to an [`Backend`].
99pub struct FunctionConfig {
100    /// The function's schema. The last column in the schema is the return type.
101    pub schema: Vec<ColumnTy>,
102    /// The behavior of the function when lookups are made on keys not currently present.
103    pub default: DefaultVal,
104    /// How to resolve FD conflicts for the function.
105    pub merge: MergeFn,
106    /// The function's name
107    pub name: String,
108    /// Whether or not subsumption is enabled for this function.
109    pub can_subsume: bool,
110}
111
112/// How defaults are computed for the given function.
113#[derive(Copy, Clone)]
114pub enum DefaultVal {
115    /// Generate a fresh UF id.
116    FreshId,
117    /// Cause an egglog-level panic if a lookup fails.
118    Fail,
119    /// Insert a constant of some kind.
120    Const(Value),
121}
122
123/// How to resolve FD conflicts for a table.
124pub enum MergeFn {
125    /// Panic if the old and new values don't match.
126    AssertEq,
127    /// Use congruence to resolve FD conflicts.
128    UnionId,
129    /// The output of a merge is determined by applying the given ExternalFunction to the result
130    /// of the argument merge functions.
131    Primitive(ExternalFunctionId, Vec<MergeFn>),
132    /// The output of a merge is determined by looking up the value for the given function and the
133    /// given arguments in the egraph.
134    Function(FunctionId, Vec<MergeFn>),
135    /// Always return the old value for the given function.
136    Old,
137    /// Always return the new value for the given function.
138    New,
139    /// Always overwrite the new value for the given function with a constant. This is more useful
140    /// as a "base case" in a more complicated merge function (e.g. one that clamps a value between
141    /// 1 and 100) than it is as a standalone merge function.
142    Const(Value),
143}

162/// A variable in a rule body / RHS, with an optional display name for

177/// A reference in a rule body or RHS: either a variable or a typed constant.
178#[derive(Clone, Debug, PartialEq, Eq, Hash)]
179pub enum QueryEntry {
180    Var(Variable),
181    Const {
182        val: Value,
183        // Constants can have a type plumbed through, particularly if they
184        // correspond to a base value constant in egglog.
185        ty: ColumnTy,
186    },
187}
188
189impl From<Variable> for QueryEntry {
190    fn from(var: Variable) -> Self {
191        QueryEntry::Var(var)
192    }

196// The `Backend` trait

199/// A backend that drives an egglog egraph.
200///
201/// Implementations: `egglog_bridge::EGraph` (reference, in-memory) and
202/// `egglog_bridge_duckdb::EGraph` (DuckDB-backed). The frontend `EGraph` in
203/// the `egglog` crate holds a `Box<dyn Backend>` and dispatches all state
204/// access through this trait.
205///
206/// ## Method correspondence
207///
208/// Each method's doc comment names the inherent method on
209/// `egglog_bridge::EGraph` that the bridge's `impl Backend` wraps. The
210/// reference backend's impl is meant to be one line per method
211/// (`fn foo(&self, ...) -> X { self.foo(...) }`). The DuckDB backend's
212/// translations are documented in `docs/backend_trait_design.md`.
213///
214/// ## Dyn-compatibility
215///
216/// All methods are object-safe. Methods that would otherwise be generic over
217/// `T: BaseValue` or `C: ContainerValue` live on the
218/// [`BaseValuePool`] / [`ContainerPool`] sub-traits, which use Any-based
219/// dynamic dispatch internally.
220///
221/// ## Bridge-only escape hatch via [`Backend::as_any`]
222///
223/// A handful of frontend constructs — most notably `TableAction::new` /
224/// `UnionAction::new` used inside `rust_rule` and `query` primitive
225/// `apply()` bodies in `src/prelude.rs` — depend on bridge-specific state
226/// (`egglog_bridge::EGraph`) that does not fit cleanly behind the trait
227/// surface. The Phase 1 design (see `docs/backend_trait_design.md`,
228/// "How `TableAction` and `UnionAction` are handled") explicitly keeps
229/// these types as bridge inherent methods rather than lifting them onto
230/// the trait.
231///
232/// To let callers reach the bridge-specific state when the backend field
233/// is `Box<dyn Backend>`, the trait exposes an `Any`-based downcast pair:
234///
235/// ```ignore
236/// // Inside a primitive that needs TableAction:
237/// let bridge = self
238///     .backend
239///     .as_any()
240///     .downcast_ref::<egglog_bridge::EGraph>()
241///     .expect("table actions are bridge-only");
242/// let action = egglog_bridge::TableAction::new(bridge, func_id);
243/// ```
244///
245/// The DuckDB backend's `as_any` returns a different concrete type
246/// (`egglog_bridge_duckdb::EGraph`); the downcast to
247/// `&egglog_bridge::EGraph` therefore fails on DuckDB. This is correct: the
248/// primitives that go through this path (`rust_rule` / `query` table-lookup
249/// helpers) are gated by [`Backend::supports_inline_table_lookups`] and
250/// are documented as unsupported on DuckDB in v1.
251///
252/// Using `&dyn Any` keeps this trait crate from having to name
253/// `egglog_bridge::EGraph` directly, which would re-introduce the
254/// `egglog-bridge ↔ egglog-backend-trait` dependency cycle that Phase 2
255/// Commit 3 deliberately broke.
256pub trait Backend: Send + Sync {
257    // -- table lifecycle ----------------------------------------------------
258
259    /// Register a function/relation/constructor and return its handle.
260    ///
261    /// Wraps `egglog_bridge::EGraph::add_table`.
262    fn add_table(&mut self, config: FunctionConfig) -> FunctionId;
263
264    /// Number of rows currently in the given function's table.
265    ///
266    /// Wraps `egglog_bridge::EGraph::table_size`.
267    fn table_size(&self, table: FunctionId) -> usize;
268
269    /// Approximate size; backends may return a fast estimate.
270    ///
271    /// Wraps `egglog_bridge::EGraph::approx_table_size`.
272    fn approx_table_size(&self, table: FunctionId) -> usize;
273
274    // -- iteration ----------------------------------------------------------
275
276    /// Iterate over every row in `table`, calling `f` on each.
277    ///
278    /// Wraps `egglog_bridge::EGraph::for_each`.
279    ///
280    /// The closure's `FunctionRow` is borrowed from a transient per-call
281    /// buffer (a `TaggedRowBuffer` in the bridge; a row cursor in DuckDB).
282    /// The HRTB lifetime `for<'r>` reflects that the row reference is
283    /// scoped to the closure invocation, not the outer `&self` borrow.
284    fn for_each(&self, table: FunctionId, f: &mut dyn for<'r> FnMut(FunctionRow<'r>)) {
285        // The default implementation cannot be written here without a wrapper
286        // because the closure types differ. Implementations should override
287        // this to avoid the boolean threading overhead; the no-default
288        // alternative would be to require both methods. We mark this as
289        // mandatory below to keep impls explicit.
290        let _ = (table, f);
291        unimplemented!(
292            "Backend impls must override for_each; the default exists only \
293             to satisfy dyn-compatibility lint chains."
294        )
295    }
296
297    /// Iterate over rows in `table`, stopping early when `f` returns `false`.
298    ///
299    /// Wraps `egglog_bridge::EGraph::for_each_while`.
300    ///
301    /// See [`Backend::for_each`] for why the closure uses an HRTB.
302    fn for_each_while(&self, table: FunctionId, f: &mut dyn for<'r> FnMut(FunctionRow<'r>) -> bool);
303
304    // -- direct access ------------------------------------------------------
305
306    /// Look up the output value associated with `key` in `func`.
307    ///
308    /// Wraps `egglog_bridge::EGraph::lookup_id`.
309    fn lookup_id(&self, func: FunctionId, key: &[Value]) -> Option<Value>;
310
311    /// Bulk-insert one or many rows across one or many functions.
312    ///
313    /// Wraps `egglog_bridge::EGraph::add_values`. (The boxed iterator is
314    /// used to keep this method dyn-compatible.)
315    fn add_values(&mut self, values: Box<dyn Iterator<Item = (FunctionId, Vec<Value>)> + '_>);
316
317    /// Add a term-shaped row: stage `(inputs ... fresh_id)` and return the
318    /// canonical id of the freshly allocated output.
319    ///
320    /// Wraps `egglog_bridge::EGraph::add_term`. On DuckDB this maps to an
321    /// `INSERT ... RETURNING` against the function's table.
322    fn add_term(&mut self, func: FunctionId, inputs: &[Value]) -> Value;
323
324    /// Stage a batch of complete rows into `table`.
325    ///
326    /// Each entry in `rows` is the full row for the table (key columns
327    /// followed by the value column, where applicable). The rows are staged
328    /// against a single internal execution state; callers should invoke
329    /// [`Backend::flush_updates`] afterward to merge them into the database.
330    ///
331    /// Unlike [`Backend::add_values`], this is scoped to a single table and
332    /// does not call `flush_updates` itself, which makes it suitable for
333    /// scenarios that interleave many table-level insertions before a final
334    /// flush (e.g. parsing an input file, scheduler match dispatch).
335    ///
336    /// The bridge's impl wraps `with_execution_state` around a loop of
337    /// `TableAction::insert` calls.
338    fn insert_rows(&mut self, table: FunctionId, rows: &[Vec<Value>]);
339
340    /// Stage a batch of constructor-style lookup-or-insert rows.
341    ///
342    /// Each entry in `rows` is a key for the constructor `table` (without the
343    /// output column); the bridge calls `TableAction::lookup` per row, which
344    /// either returns the existing output or allocates a fresh id. As with
345    /// [`Backend::insert_rows`], callers should invoke
346    /// [`Backend::flush_updates`] afterward.
347    ///
348    /// The bridge's impl wraps `with_execution_state` around a loop of
349    /// `TableAction::lookup` calls.
350    fn lookup_constructor_rows(&mut self, table: FunctionId, rows: &[Vec<Value>]);
351
352    /// Get the canonical representative of `val` according to `ty`.
353    ///
354    /// For `ColumnTy::Id` this is the union-find canonicalization. For
355    /// `ColumnTy::Base(_)` it returns `val` unchanged.
356    ///
357    /// Wraps `egglog_bridge::EGraph::get_canon_repr`.
358    fn get_canon_repr(&self, val: Value, ty: ColumnTy) -> Value;
359
360    /// Allocate a fresh egraph id (counter increment, returned as a `Value`).
361    ///
362    /// Wraps `egglog_bridge::EGraph::fresh_id`.
363    fn fresh_id(&mut self) -> Value;
364
365    /// Remove every row from the given function's table.
366    ///
367    /// Wraps `egglog_bridge::EGraph::clear_table`.
368    fn clear_table(&mut self, func: FunctionId);
369
370    /// Access the backend's [`BaseValues`] registry directly.
371    ///
372    /// Wraps `egglog_bridge::EGraph::base_values`. This returns the concrete
373    /// `egglog_core_relations::BaseValues` (re-exported here) so callers can
374    /// use the generic-over-`T` `get::<T>` / `unwrap::<T>` sugar that the
375    /// dyn-friendly [`BaseValuePool`] does not expose.
376    fn base_values(&self) -> &BaseValues;
377
378    /// Run `f` against a fresh execution state, dyn-compatible form.
379    ///
380    /// Wraps `egglog_bridge::EGraph::with_execution_state`. The generic,
381    /// result-returning ergonomic wrapper is provided as the inherent
382    /// [`Backend::with_execution_state`] method on `dyn Backend`; this
383    /// erased form exists so the trait stays object-safe.
384    fn with_execution_state_dyn(&self, f: &mut dyn FnMut(&mut ExecutionState<'_>));
385
386    /// Type-erased access to the backend's action registry handle.
387    ///
388    /// Wraps `egglog_bridge::EGraph::action_registry`, which returns a
389    /// `&Arc<RwLock<ActionRegistry>>`. `ActionRegistry` is defined in the
390    /// `egglog-bridge` crate, which depends on this crate; naming it here
391    /// would re-introduce the dependency cycle that Phase 2 deliberately
392    /// broke. So the object-safe form erases the handle to `&dyn Any`, and
393    /// the generic inherent [`Backend::action_registry`] downcasts it back
394    /// to the caller-inferred concrete registry type.
395    ///
396    /// The `&dyn Any` is expected to be an `Arc<RwLock<ActionRegistry>>`.
397    /// Backends without an action registry (e.g. duckdb) panic.
398    fn action_registry_any(&self) -> &(dyn Any + Send + Sync);
399
400    // -- rule management ----------------------------------------------------
401
402    /// Begin building a new rule. Returns a builder whose lifetime is tied to
403    /// `&mut self`. Callers populate the builder via [`RuleBuilderOps`] and
404    /// finalize with [`RuleBuilderOps::build`].
405    ///
406    /// Wraps `egglog_bridge::EGraph::new_rule`.
407    fn new_rule<'a>(&'a mut self, desc: &str, seminaive: bool) -> Box<dyn RuleBuilderOps + 'a>;
408
409    /// Drop a registered rule. The handle becomes invalid.
410    ///
411    /// Wraps `egglog_bridge::EGraph::free_rule`.
412    fn free_rule(&mut self, id: RuleId);
413
414    /// Run one iteration of the given rule set. Returns timing and change
415    /// counts; the database may have been modified even if `changed` is
416    /// false (e.g. timestamps advanced).
417    ///
418    /// Wraps `egglog_bridge::EGraph::run_rules`.
419    fn run_rules(&mut self, rules: &[RuleId]) -> Result<IterationReport>;
420
421    /// Drain staged inserts and run a rebuild pass if the UF changed.
422    /// Returns whether the database changed.
423    ///
424    /// Wraps `egglog_bridge::EGraph::flush_updates`.
425    fn flush_updates(&mut self) -> bool;
426
427    // -- primitives ---------------------------------------------------------
428
429    /// Register a user-defined primitive (`ExternalFunction`).
430    ///
431    /// Wraps `egglog_bridge::EGraph::register_external_func`.
432    ///
433    /// On DuckDB, primitives that synchronously call back into table state
434    /// (e.g. `TableAction::lookup` in the apply body) are not supported in
435    /// v1. See [`Backend::supports_inline_table_lookups`].
436    fn register_external_func(
437        &mut self,
438        func: Box<dyn ExternalFunction + 'static>,
439    ) -> ExternalFunctionId;
440
441    /// Drop a user-defined primitive.
442    ///
443    /// Wraps `egglog_bridge::EGraph::free_external_func`.
444    fn free_external_func(&mut self, func: ExternalFunctionId);
445
446    /// Register a deferred-panic primitive that stops rule execution when
447    /// invoked. The returned id can be used as a fallback target in
448    /// `lookup_with_fallback` and similar.
449    ///
450    /// Wraps `egglog_bridge::EGraph::new_panic`.
451    fn new_panic(&mut self, message: String) -> ExternalFunctionId;
452
453    /// Evaluate a registered primitive on the given (already-interned) argument
454    /// `Value`s, returning its result, or `None` if the primitive fails
455    /// (e.g. `!=` of equal arguments, or a guard that does not hold).
456    ///
457    /// This is the **backend-agnostic primitive entry point**. It lets a
458    /// backend evaluate a primitive *inline* (e.g. inside a DBSP map/filter
459    /// operator, or a host-side join loop) without owning or borrowing a
460    /// `core_relations::ExecutionState`, which is the type
461    /// [`ExternalFunction::invoke`] requires.
462    ///
463    /// ## Contract
464    ///
465    /// - `args` are interned `Value`s in the backend's value space (same
466    ///   representation `for_each` / `lookup_id` use).
467    /// - The result, if any, is an interned `Value` in that same space.
468    /// - For backends that evaluate primitives against an execution state
469    ///   (reference, feldera), this is `with_execution_state(|st|
470    ///   st.call_external_func(id, args))`.
471    /// - Pure primitives (comparisons, arithmetic, string ops) do not touch
472    ///   the database, so they evaluate identically regardless of which
473    ///   backend hosts them. Primitives that reenter table state are only
474    ///   well-defined on backends whose
475    ///   [`Backend::supports_inline_table_lookups`] is `true`.
476    fn eval_prim(&self, id: ExternalFunctionId, args: &[Value]) -> Option<Value>;
477
478    // -- typed value handles (sub-traits) -----------------------------------
479
480    /// Access the backend's [`BaseValuePool`] for typed base-value queries.
481    ///
482    /// Wraps `egglog_bridge::EGraph::base_values` (returns `&BaseValues`,
483    /// which implements this sub-trait either directly or via a thin shim
484    /// the bridge provides).
485    fn base_value_pool(&self) -> &dyn BaseValuePool;
486
487    /// Mutable access to the [`BaseValuePool`]. Used to register new
488    /// `BaseValue` types.
489    ///
490    /// Wraps `egglog_bridge::EGraph::base_values_mut`.
491    fn base_value_pool_mut(&mut self) -> &mut dyn BaseValuePool;
492
493    /// Access the backend's [`ContainerPool`].
494    ///
495    /// Wraps `egglog_bridge::EGraph::container_values`.
496    ///
497    /// On DuckDB this returns an empty stub (see [`ContainerPool`]'s docs).
498    /// All container-using egglog programs are gated out of DuckDB by the
499    /// existing `program_supports_proofs` check, so the stub is never
500    /// reached in practice.
501    fn container_pool(&self) -> &dyn ContainerPool;
502
503    /// Mutable access to the [`ContainerPool`].
504    ///
505    /// Wraps `egglog_bridge::EGraph::container_values_mut`.
506    fn container_pool_mut(&mut self) -> &mut dyn ContainerPool;
507
508    /// Build a [`QueryEntry`] constant for a base value of dynamic type.
509    ///
510    /// `value` is the interned `Value`; `ty` is the base-value-type id
511    /// returned by `BaseValuePool::register_type` /
512    /// `BaseValuePool::get_ty_by_type_id`.
513    ///
514    /// Wraps `egglog_bridge::EGraph::base_value_constant`. (The bridge's
515    /// inherent method is generic over `T: BaseValue`; the dyn-friendly form
516    /// takes the already-interned `Value` and the `BaseValueId`. A generic
517    /// helper that wraps this is provided below
518    /// (`base_value_constant<T>`).)
519    fn base_value_constant_dyn(&self, value: Value, ty: BaseValueId) -> QueryEntry;
520
521    // -- capability flags ---------------------------------------------------
522
523    /// Whether this backend's user-defined primitives can synchronously
524    /// call back into table state during their `apply()` body.
525    ///
526    /// Reference backend: `true`. DuckDB: `false` (primitives run inside
527    /// DuckDB VScalar UDFs and cannot reenter the database).
528    ///
529    /// Callers that need `rust_rule` / `query` should gate on this flag.
530    fn supports_inline_table_lookups(&self) -> bool;
531
532    /// Whether this backend supports the `subsume` action and the
533    /// `is_subsumed` filter on table atoms.
534    ///
535    /// Reference backend: `true`. DuckDB: `false` in v1 — the trait
536    /// surfaces `subsume` on [`RuleBuilderOps`] but the DuckDB impl returns
537    /// an error when called.
538    fn supports_subsumption(&self) -> bool;
539
540    /// Whether this backend supports `MergeFn::Function` and
541    /// `MergeFn::Primitive`.
542    ///
543    /// Reference backend: `true`. DuckDB: `false` in v1.
544    fn supports_complex_merge(&self) -> bool;
545
546    /// Whether this backend supports `Vec` / `Set` / `Map` / `MultiSet`
547    /// container sorts.
548    ///
549    /// Reference backend: `true`. DuckDB: `false` (container sorts are
550    /// excluded from DuckDB test combos by the existing `supports_proofs`
551    /// gate; see `docs/backend_trait_inventory.md` Section 6.3).
552    fn supports_containers(&self) -> bool;
553
554    // -- diagnostics --------------------------------------------------------
555
556    /// Set the verbosity of the per-rule-iteration timing report.
557    ///
558    /// Wraps `egglog_bridge::EGraph::set_report_level`.
559    fn set_report_level(&mut self, level: ReportLevel);
560
561    /// Dump the database state to the `log::info!` channel (debug only).
562    ///
563    /// Wraps `egglog_bridge::EGraph::dump_debug_info`.
564    fn dump_debug_info(&self);
565
566    // -- cloning ------------------------------------------------------------
567
568    /// Produce a deep clone of this backend.
569    ///
570    /// The frontend uses this for push/pop snapshot support. The reference
571    /// backend derives `Clone`, so its impl is a one-liner
572    /// (`Box::new(self.clone())`). The DuckDB backend will need a bespoke
573    /// implementation (database snapshot / replay buffer); see
574    /// `docs/backend_trait_design.md` for the chosen strategy.
575    fn clone_boxed(&self) -> Box<dyn Backend>;
576
577    // -- bridge-only escape hatch ------------------------------------------
578
579    /// Return `&self` as `&dyn Any`, enabling callers to downcast to the
580    /// concrete backend type (e.g. `&egglog_bridge::EGraph`).
581    ///
582    /// This is the supported path for invoking bridge-only inherent
583    /// methods such as `TableAction::new` / `UnionAction::new` from
584    /// frontend code whose `backend` field is a `Box<dyn Backend>`. See the
585    /// trait-level documentation for the rationale (Any-based downcast
586    /// keeps the trait crate free of `egglog-bridge` dependencies, avoiding
587    /// a dependency cycle).
588    ///
589    /// Implementations are expected to be one-liners: `fn as_any(&self) ->
590    /// &dyn Any { self }`. The trait can't supply a default body because
591    /// `dyn Backend` is not `Sized` and `&dyn Any` requires
592    /// `Self: 'static + Sized` — but every concrete backend type satisfies
593    /// both, so each impl trivially provides the body.
594    fn as_any(&self) -> &dyn Any;
595
596    /// Mutable counterpart of [`Backend::as_any`]. Implementations are
597    /// expected to return `self`.
598    fn as_any_mut(&mut self) -> &mut dyn Any;
599}
600
601impl Clone for Box<dyn Backend> {
602    fn clone(&self) -> Self {
603        self.clone_boxed()
604    }
605}

607impl dyn Backend {
608    /// Run `f` against a fresh execution state and return its result.
609    ///
610    /// Ergonomic, generic-over-`R` wrapper around the object-safe
611    /// [`Backend::with_execution_state_dyn`]. The result is threaded back
612    /// out through a captured slot rather than `Box<dyn Any>`, so `R` does
613    /// not need to be `'static`. Mirrors
614    /// `egglog_bridge::EGraph::with_execution_state`.
615    pub fn with_execution_state<R>(&self, f: impl FnOnce(&mut ExecutionState<'_>) -> R) -> R {
616        let mut f = Some(f);
617        let mut out: Option<R> = None;
618        self.with_execution_state_dyn(&mut |es| {
619            let f = f.take().expect("with_execution_state closure called once");
620            out = Some(f(es));
621        });
622        out.expect("with_execution_state_dyn must invoke its closure exactly once")
623    }
624
625    /// A handle to the live action registry for this backend.
626    ///
627    /// Generic ergonomic wrapper around [`Backend::action_registry_any`].
628    /// `R` is the concrete registry type (`egglog_bridge::ActionRegistry`),
629    /// inferred at the call site so this crate never has to name it (which
630    /// would re-introduce the `egglog-bridge -> egglog-backend-trait`
631    /// dependency cycle). Mirrors `egglog_bridge::EGraph::action_registry`.
632    ///
633    /// # Panics
634    ///
635    /// Panics if the backend has no action registry (e.g. duckdb) or if the
636    /// erased handle is not an `Arc<RwLock<R>>`.
637    pub fn action_registry<R: Any + Send + Sync>(&self) -> &std::sync::Arc<std::sync::RwLock<R>> {
638        self.action_registry_any()
639            .downcast_ref::<std::sync::Arc<std::sync::RwLock<R>>>()
640            .expect("action_registry: backend has no action registry of the requested type")
641    }
642}

644// ---------------------------------------------------------------------------
645// `RuleBuilderOps` — mirrors `egglog_bridge::RuleBuilder` one-for-one
646// ---------------------------------------------------------------------------
647
648/// A lazily-computed failure message for RHS lookups / external calls.
649///
650/// The message is only built if the lookup or call actually fails at
651/// runtime. Deferring it matters because the usual message form is
652/// `format!("{span}: ...")`, and formatting a `Span` calls
653/// `SrcFile::get_location`, which scans the source-file prefix. Building it
654/// eagerly for every action in every rule is `O(rules * file)` — quadratic
655/// when many generated rules (term/proof encoding) are compiled against a
656/// large source file. Boxed so the trait stays dyn-compatible.
657pub type PanicMsg = Box<dyn FnOnce() -> String + Send>;
658
659/// Operations on an in-progress rule.
660///
661/// This trait mirrors the public methods on
662/// `egglog_bridge::RuleBuilder`. The bridge's impl is a trivial newtype
663/// passthrough; the DuckDB impl accumulates calls into its internal
664/// `duck::Rule` data IR and submits to `compile_rule` on
665/// [`RuleBuilderOps::build`].
666///
667/// ## Variable creation
668///
669/// `new_var` and `new_var_named` allocate new variables. The returned
670/// `QueryEntry` can be passed back to body atoms / actions. Each `QueryEntry`
671/// carries its [`ColumnTy`] for runtime arity / type checking.
672///
673/// ## Unsupported on DuckDB
674///
675/// - [`RuleBuilderOps::subsume`][]: error.
676/// - Complex `MergeFn::Function` / `MergeFn::Primitive` referenced via
677///   `query_prim`: error at `build()` time when the rule references such a
678///   merge.
679pub trait RuleBuilderOps {
680    /// Bind a new variable of the given type.
681    ///
682    /// Wraps `RuleBuilder::new_var`.
683    fn new_var(&mut self, ty: ColumnTy) -> QueryEntry;
684
685    /// Bind a new variable of the given type with a display name (eases
686    /// debugging).
687    ///
688    /// Wraps `RuleBuilder::new_var_named`.
689    fn new_var_named(&mut self, ty: ColumnTy, name: &str) -> QueryEntry;
690
691    /// Add a table body atom. The final entry is the function's return
692    /// value. When `is_subsumed` is `Some`, the atom is constrained to rows
693    /// with that subsumption bit.
694    ///
695    /// Wraps `RuleBuilder::query_table`.
696    fn query_table(
697        &mut self,
698        func: FunctionId,
699        entries: &[QueryEntry],
700        is_subsumed: Option<bool>,
701    ) -> Result<()>;
702
703    /// Add a primitive body atom. The final entry is the return value.
704    ///
705    /// Wraps `RuleBuilder::query_prim`.
706    fn query_prim(
707        &mut self,
708        func: ExternalFunctionId,
709        entries: &[QueryEntry],
710        ret_ty: ColumnTy,
711    ) -> Result<()>;
712
713    /// Call an external function in the RHS, panicking with `panic_msg` on
714    /// failure. Returns the result variable.
715    ///
716    /// Wraps `RuleBuilder::call_external_func`. `panic_msg` is a [`PanicMsg`]
717    /// closure, evaluated only if the call actually fails.
718    fn call_external_func(
719        &mut self,
720        func: ExternalFunctionId,
721        args: &[QueryEntry],
722        ret_ty: ColumnTy,
723        panic_msg: PanicMsg,
724    ) -> QueryEntry;
725
726    /// RHS: look up the value of `func(entries)`, with the function's
727    /// configured default behavior on miss.
728    ///
729    /// Wraps `RuleBuilder::lookup`. `panic_msg` is a [`PanicMsg`] closure
730    /// (evaluated only on miss when the function is `DefaultVal::Fail`).
731    fn lookup(
732        &mut self,
733        func: FunctionId,
734        entries: &[QueryEntry],
735        panic_msg: PanicMsg,
736    ) -> QueryEntry;
737
738    /// RHS: subsume the row keyed by `entries` in `func`.
739    ///
740    /// Wraps `RuleBuilder::subsume`.
741    ///
742    /// **DuckDB**: errors at this call site. Programs that need subsume
743    /// must use the reference backend in v1.
744    fn subsume(&mut self, func: FunctionId, entries: &[QueryEntry]) -> Result<()>;
745
746    /// RHS: set `func(entries[..n-1])` to `entries[n-1]`.
747    ///
748    /// Wraps `RuleBuilder::set`.
749    fn set(&mut self, func: FunctionId, entries: &[QueryEntry]);
750
751    /// RHS: remove the row keyed by `entries` from `func`.
752    ///
753    /// Wraps `RuleBuilder::remove`.
754    fn remove(&mut self, func: FunctionId, entries: &[QueryEntry]);
755
756    /// RHS: merge two values in the union-find.
757    ///
758    /// Wraps `RuleBuilder::union`.
759    fn union(&mut self, l: QueryEntry, r: QueryEntry);
760
761    /// RHS: panic with the given message.
762    ///
763    /// Wraps `RuleBuilder::panic`.
764    fn panic(&mut self, message: String);
765
766    /// Register a deferred-panic external function on the underlying egraph
767    /// and return its id. Used (e.g. by `unstable-fn`) to bake a panic id
768    /// into a wrapped function's call site.
769    ///
770    /// Wraps `RuleBuilder::new_panic`. Default panics for backends that do
771    /// not support it; the bridge forwards to its `RuleBuilder`.
772    fn new_panic(&mut self, _message: String) -> ExternalFunctionId {
773        unimplemented!("new_panic is not supported on this backend")
774    }
775
776    /// Skip tree-decomposition during query planning for this rule
777    /// (the `:no-decomp` option / `--no-decomp`). Default no-op for
778    /// backends that don't decompose (e.g. duckdb); the bridge
779    /// forwards it to its `RuleBuilder`.
780    fn set_no_decomp(&mut self, _no_decomp: bool) {}
781
782    /// Finalize the rule. Returns the registered [`RuleId`].
783    ///
784    /// Wraps `RuleBuilder::build`. The DuckDB impl hands the accumulated
785    /// `duck::Rule` IR off to `compile_rule` and inserts it into the
786    /// backend's rule registry. If any accumulated call referenced a
787    /// feature the backend does not support (e.g. subsume on DuckDB), this
788    /// is where the error surfaces.
789    fn build(self: Box<Self>) -> Result<RuleId>;
790
791    /// Consume the builder and check whether its accumulated body
792    /// matches at least one assignment in the current database.
793    /// Actions accumulated in the builder are ignored. The bridge
794    /// backend's default implementation panics; the DuckDB backend
795    /// overrides this to compile the body to SQL and run it.
796    /// Used by `(check …)` on the DuckDB path; the bridge keeps its
797    /// existing external-func side-channel approach in `check_facts`.
798    fn build_check(self: Box<Self>) -> Result<bool> {
799        Err(anyhow::anyhow!(
800            "build_check is not implemented for this backend"
801        ))
802    }
803
804    /// Rename an already-registered primitive on the backend so that
805    /// the per-id name reflects the call-site's concrete type. Used
806    /// to disambiguate built-in primitives whose egglog name is
807    /// overloaded across sorts (e.g. `^` is XOR for i64 and POWER for
808    /// f64; `+` for String is concat). The bridge backend doesn't
809    /// need this (it dispatches via the typed `ExternalFunction`
810    /// directly), so the default is a no-op. The duck rule-builder
811    /// routes the call through the egraph it already holds mutably.
812    fn rename_prim(&mut self, _id: ExternalFunctionId, _name: String) {}
813}

816// `BaseValuePool` — dyn-compatible base-value registry
817// ---------------------------------------------------------------------------
818
819/// A registry for base-value types, exposed through dynamic dispatch so it
820/// fits inside `dyn Backend`.
821///
822/// Generic-over-`T: BaseValue` helpers are provided at the bottom of this
823/// module (`pool_get<T>`, `pool_unwrap<T>`, `pool_register_type<T>`,
824/// `pool_get_ty<T>`); they wrap the dyn methods below.
825///
826/// The bridge's impl forwards to `egglog_core_relations::BaseValues`
827/// directly. The DuckDB impl maintains its own `BaseValues`-shaped registry
828/// in-process; entries for `i64`/`f64`/`bool`/`String`/`()` are encoded
829/// inline into SQL columns where possible, and exotic types fall back to
830/// an in-memory intern table identical to the bridge's.
831pub trait BaseValuePool: Send + Sync {
832    /// Register a new base-value type using its `TypeId` plus a factory
833    /// that constructs a fresh typed intern table for the type.
834    ///
835    /// The factory is invoked at most once: if a type with the same
836    /// `TypeId` is already registered, the existing [`BaseValueId`] is
837    /// returned and the factory is dropped.
838    ///
839    /// Callers typically use [`pool_register_type`], the generic-over-`T`
840    /// helper below, which constructs the appropriate factory and threads
841    /// it through.
842    ///
843    /// Wraps `BaseValues::register_type_dyn` on the bridge side.
844    fn register_type_dyn(
845        &mut self,
846        type_id: TypeId,
847        factory: Box<dyn FnOnce() -> Box<dyn DynamicInternTable>>,
848    ) -> BaseValueId;
849
850    /// Look up the `BaseValueId` for a registered base-value type by its
851    /// Rust `TypeId`.
852    ///
853    /// Wraps `BaseValues::get_ty_by_id`.
854    fn get_ty_by_type_id(&self, type_id: TypeId) -> BaseValueId;
855
856    /// Intern an opaque (already-boxed) base value, returning its `Value`
857    /// handle. The `Box<dyn Any>` must contain a value of the type
858    /// previously registered as `ty`.
859    ///
860    /// Wraps the dyn dispatch over `BaseValues::get<P>`. Concrete-`T`
861    /// callers should prefer `pool_get<T>` below, which special-cases
862    /// `T::MAY_UNBOX` and avoids boxing.
863    fn intern_dyn(&self, ty: BaseValueId, value: Box<dyn Any + Send + Sync>) -> Value;
864
865    /// Extract a base value of the registered type `ty` from `val`. The
866    /// returned `Box<dyn Any>` holds a value of the same type that was
867    /// previously interned with `intern_dyn`.
868    ///
869    /// Wraps the dyn dispatch over `BaseValues::unwrap<P>`. Concrete-`T`
870    /// callers should prefer `pool_unwrap<T>` below.
871    fn unwrap_dyn(&self, ty: BaseValueId, val: Value) -> Box<dyn Any + Send + Sync>;
872
873    /// True iff a base-value type with the given `TypeId` is registered.
874    fn has_ty(&self, type_id: TypeId) -> bool;
875}
876
877/// Generic helper: register a `T: BaseValue` with the pool and return its
878/// [`BaseValueId`]. Equivalent to `BaseValues::register_type::<T>`.
879///
880/// This sits outside the trait so the trait remains dyn-compatible. It
881/// builds the typed intern-table factory closure that
882/// [`BaseValuePool::register_type_dyn`] requires.
883pub fn pool_register_type<T: BaseValue>(pool: &mut dyn BaseValuePool) -> BaseValueId {
884    pool.register_type_dyn(
885        TypeId::of::<T>(),
886        Box::new(|| egglog_core_relations::new_intern_table::<T>()),
887    )
888}
889
890/// Generic helper: look up the `BaseValueId` for `T: BaseValue`.
891pub fn pool_get_ty<T: BaseValue>(pool: &dyn BaseValuePool) -> BaseValueId {
892    pool.get_ty_by_type_id(TypeId::of::<T>())
893}
894
895/// Generic helper: intern a typed base value into the pool, returning its
896/// `Value`. Mirrors `BaseValues::get<T>`. Honors `T::MAY_UNBOX` for
897/// inline-encodable types.
898pub fn pool_get<T: BaseValue>(pool: &dyn BaseValuePool, value: T) -> Value {
899    if T::MAY_UNBOX
900        && let Some(v) = value.try_box()
901    {
902        return v;
903    }
904    let ty = pool_get_ty::<T>(pool);
905    pool.intern_dyn(ty, Box::new(value))
906}
907
908/// Generic helper: extract a typed base value of `T` from a `Value`.
909/// Mirrors `BaseValues::unwrap<T>`. Honors `T::MAY_UNBOX`.
910///
911/// # Panics
912///
913/// Panics if `val` does not correspond to a value of type `T` previously
914/// interned in `pool` (matching the bridge's existing semantics).
915pub fn pool_unwrap<T: BaseValue>(pool: &dyn BaseValuePool, val: Value) -> T {
916    if T::MAY_UNBOX
917        && let Some(p) = T::try_unbox(val)
918    {
919        return p;
920    }
921    let ty = pool_get_ty::<T>(pool);
922    let boxed = pool.unwrap_dyn(ty, val);
923    *boxed
924        .downcast::<T>()
925        .expect("BaseValuePool::unwrap_dyn returned wrong type")
926}

929// `ContainerPool` — stub-friendly container registry
930// ---------------------------------------------------------------------------
931
932/// A registry for container values (`Vec` / `Set` / `Map` / `MultiSet` /
933/// user-defined `ContainerValue` impls).
934///
935/// ## Two implementations
936///
937/// - **Reference backend**: delegates to
938///   `egglog_core_relations::ContainerValues`. All methods succeed; the
939///   pool participates in the EGraph's rebuild loop.
940/// - **DuckDB backend**: an empty stub. All accessor methods return `None`
941///   / empty iterators; all mutators return errors. This is safe because
942///   the term-encoding gate (`program_supports_proofs` in
943///   `src/proofs/proof_encoding_helpers.rs`) excludes every container-using
944///   program from DuckDB's test combos. The stub's error path is a
945///   defensive measure for programmer error, never a routine code path.
946///
947/// ## Why not just generic-over-`C`?
948///
949/// `register_val<C>` / `for_each<C>` are generic over `C: ContainerValue`,
950/// which is incompatible with `dyn Backend`. We expose Any-based dynamic
951/// dispatch here; concrete-`C` sugar is implemented as free functions in a
952/// future commit (the call sites in `src/lib.rs:1924`, `serialize.rs`,
953/// `extract.rs` will each adopt the dyn API directly because they already
954/// thread a `TypeId` through).
955pub trait ContainerPool: Send + Sync {
956    /// True iff a container type with the given `TypeId` is registered.
957    fn has_container_type(&self, type_id: TypeId) -> bool;
958
959    /// True iff this backend supports containers at all.
960    ///
961    /// Reference: `true`. DuckDB: `false`. Provided as a convenience so
962    /// callers don't need to consult [`Backend::supports_containers`].
963    fn enabled(&self) -> bool;
964
965    /// Look up the container value associated with `val`. Returns `None`
966    /// when no entry is registered.
967    ///
968    /// The returned `Box<dyn Any>` holds an instance of the container's
969    /// registered Rust type.
970    ///
971    /// Wraps `ContainerValues::get_val`. On DuckDB always returns `None`.
972    fn get_dyn(&self, ty: TypeId, val: Value) -> Option<Box<dyn Any + Send + Sync>>;
973
974    /// Register a Rust container value, returning a fresh `Value` handle.
975    ///
976    /// On DuckDB this returns an error (containers unsupported).
977    fn register_val_dyn(&mut self, ty: TypeId, value: Box<dyn Any + Send + Sync>) -> Result<Value>;
978
979    /// Iterate (id, container) pairs for all registered values of a
980    /// container type. The callback receives the container as a
981    /// `&dyn Any` of the registered concrete type.
982    ///
983    /// Wraps `ContainerValues::for_each<C>`. On DuckDB this is a no-op
984    /// (the stub registry is empty).
985    fn for_each_dyn(&self, ty: TypeId, f: &mut dyn FnMut(Value, &dyn Any));
986
987    /// Number of registered values of the given container type.
988    fn size(&self, ty: TypeId) -> usize;
989}
990
991/// Generic helper: register a `C: ContainerValue` with the pool, returning
992/// its `Value`. On DuckDB this returns an error.
993pub fn container_register_val<C: ContainerValue>(
994    pool: &mut dyn ContainerPool,
995    value: C,
996) -> Result<Value> {
997    pool.register_val_dyn(TypeId::of::<C>(), Box::new(value))
998}
