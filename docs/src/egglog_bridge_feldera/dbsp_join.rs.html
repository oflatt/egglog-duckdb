1//! DBSP-backed body join for the Feldera backend (#23 Stage C complete).
2//!
3//! This module runs a rule's **relational body join on DBSP's dataflow engine**
4//! — the paper's core technical contribution. The persistent per-rule circuit
5//! ([`PersistentJoin`]) is the ONLY join path: there is no host nested-loop
6//! fallback. A genuinely-ineligible rule PANICS (see [`plan_join`]).
7//!
8//! ## What runs on DBSP
9//!
10//! [`PersistentJoin`] builds a **non-recursive** DBSP circuit ONCE per rule that
11//! computes the join of the body's table atoms (multi-atom, left-deep), with
12//! `!=` guards applied as DBSP `filter` operators *inside* the dataflow. It is
13//! fed only the per-relation signed DELTA each iteration; the incremental join +
14//! integral do the seminaive bookkeeping and retraction natively. The circuit's
15//! output is a z-set of **binding rows** ([`BindRow`], a [`JOIN_WIDTH`]-wide
16//! [`TupRow`]) holding the rule's body variables in a fixed canonical order.
17//!
18//! ## Pure body prims run ON-CIRCUIT
19//!
20//! Value-computing body prims are evaluated on-circuit through the shared
21//! [`crate::PrimEngine`] (an `Arc<Mutex<Database>>` captured into the circuit
22//! closures), so the whole body join — guards AND value prims — runs on DBSP.
23//! Recognized rep-comparison prims are inlined without a lock; everything else
24//! is evaluated by re-running the REAL prim under the engine lock (the call-prim
25//! path). Only head application (`set`/`delete`/`lookup`/`union` + FD-merge)
26//! stays host-side (`interpret::run_iteration` / `apply_head`).
27//!
28//! ## Eligibility / the row-width cap
29//!
30//! DBSP rows must be fixed-arity `DBData` (rkyv-archivable); we use a uniform
31//! [`JOIN_WIDTH`]-wide [`TupRow`]. A rule is eligible (does not panic) iff:
32//!   - it has at least one table atom (atom-less rules are fired once by the
33//!     caller, not planned here);
34//!   - every table atom has arity <= [`JOIN_WIDTH`];
35//!   - the rule's body uses <= [`JOIN_WIDTH`] distinct variables (binding row);
36//!   - every body primitive is PURE. A recognized rep-comparison prim (`!=`,
37//!     `bool-!=`, `or`, `guard`, `ordering-min/max`) is inlined when its operand
38//!     columns support the relevant rep-arithmetic (ORDERING prims require `Id`
39//!     columns; EQUALITY prims also accept the injectively-interned base types
40//!     `bool`/`i64`/`string`). A pure prim that is NOT rep-inlinable
41//!     (base-value `ordering-min/max`, f64 `!=`, or any unrecognized value prim
42//!     like `+`/`int-div`/`string-concat`) is NOT rejected — it ROUTES to the
43//!     on-circuit call-prim engine, which evaluates the real prim semantics on
44//!     actual values. Only an IMPURE prim (or a shape above the row-width cap)
45//!     makes the rule ineligible — and that PANICS.
46

56/// Max distinct body variables / atom columns the DBSP join supports (the
57/// fixed-arity `DBData` row width).
58///
59/// Stage C widens this from 10 (dbsp's stock [`dbsp::utils::Tup10`]) to 32 via a
60/// custom [`TupRow`] declared with dbsp's `declare_tuples!` macro, because the
61/// persistent circuit is now the ONLY join path: it must cover the WIDEST
62/// feldera-supported rule. A suite-wide survey put the peak among PASSING
63/// programs at ~23 distinct body variables (`cykjson` `check_facts`, the
64/// `const-prop` value rules — the latter also materialize on-circuit prim
65/// outputs into binding columns), so 32 gives headroom. A rule wider than this
66/// PANICS as ineligible (`plan_join`'s "too many vars" reject); the only suite
67/// program that trips it is `luminal-llama`'s `@rebuild_rule34` (35 vars), which
68/// is independently unsupported (it fails the frontend with an unbound-global
69/// error regardless of the backend). Widening to cover it (48) regressed the
70/// `math-microbenchmark` N=11 run to OOM (50% larger rows through the saturation
71/// z-sets), so 32 is the chosen ceiling: it covers every program that actually
72/// completes. Cost: a binding row is `32 * 4 = 128` bytes (vs 40 at width 10).
73/// Pulling in dbsp's DBData derive stack (paste/rkyv/size_of/serde/derive_more)
74/// is the one-time dependency price for a row wider than `Tup10`.
75pub const JOIN_WIDTH: usize = 32;

190/// Build a fixed-width row from a slice of column values (0-padded to
191/// [`JOIN_WIDTH`]). Slices longer than [`JOIN_WIDTH`] are rejected upstream by
192/// [`plan_join`].
193fn pack_row(vals: &[u32]) -> BindRow {
194    let mut a = [0u32; JOIN_WIDTH];
195    for (i, v) in vals.iter().enumerate() {
196        a[i] = *v;
197    }
198    arr_to_row(a)
199}

230/// Rep-arithmetic kind of an atom-bound column, gating which inlined prim
231/// operations are correct on its interned `u32` rep. See `plan_join`'s
232/// "Correctness gating" docs.
233#[derive(Clone, Copy, PartialEq, Eq, Debug)]
234enum RepKind {
235    /// `Id` (union-find) column: rep order = id order, so ordering AND equality
236    /// are both valid.
237    Ordering,
238    /// bool / i64 / string column: injective interning makes equality / `!=`
239    /// valid, but rep order ≠ value order so ordering is NOT inlinable.
240    Equality,
241}

243/// A pure VALUE expression over a binding row, built from inlined value prims.
244/// Evaluates to a `u32` (an interned rep). Only constructed when every leaf is
245/// a binding-row column whose type makes rep-arithmetic provably correct (an
246/// `Id`/union-find column, or — for equality only — a bool column).
247#[derive(Clone, Debug)]
248enum PureExpr {
249    /// A binding-row column (an atom-bound variable).
250    Col(usize),
251    /// A literal rep.
252    Const(u32),
253    /// `ordering-min(a, b)` = numeric min of the two reps.
254    Min(Box<PureExpr>, Box<PureExpr>),
255    /// `ordering-max(a, b)` = numeric max of the two reps.
256    Max(Box<PureExpr>, Box<PureExpr>),
257}
258
259/// A pure BOOLEAN condition over a binding row, built from inlined guard/bool
260/// prims. Lowered to a DBSP `filter` once all its leaf columns are bound.
261#[derive(Clone, Debug)]
262enum Cond {
263    /// `a != b` on reps (from `!=` and `bool-!=`).
264    Ne(PureExpr, PureExpr),
265    /// `a == b` on reps (from an `(= (ordering-* a b) c)` assert).
266    Eq(PureExpr, PureExpr),
267    /// Disjunction (from `or`).
268    Or(Vec<Cond>),
269}
270
271impl PureExpr {
272    /// Evaluate against a binding row.
273    fn eval(&self, row: &BindRow) -> u32 {
274        match self {
275            PureExpr::Col(c) => get_col(row, *c),
276            PureExpr::Const(v) => *v,
277            PureExpr::Min(a, b) => a.eval(row).min(b.eval(row)),
278            PureExpr::Max(a, b) => a.eval(row).max(b.eval(row)),
279        }
280    }
281    /// Binding-row columns this expression reads.
282    fn cols(&self, out: &mut Vec<usize>) {
283        match self {
284            PureExpr::Col(c) => out.push(*c),
285            PureExpr::Const(_) => {}
286            PureExpr::Min(a, b) | PureExpr::Max(a, b) => {
287                a.cols(out);
288                b.cols(out);
289            }
290        }
291    }
292}
293
294impl Cond {
295    fn eval(&self, row: &BindRow) -> bool {
296        match self {
297            Cond::Ne(a, b) => a.eval(row) != b.eval(row),
298            Cond::Eq(a, b) => a.eval(row) == b.eval(row),
299            Cond::Or(cs) => cs.iter().any(|c| c.eval(row)),
300        }
301    }
302    fn cols(&self, out: &mut Vec<usize>) {
303        match self {
304            Cond::Ne(a, b) | Cond::Eq(a, b) => {
305                a.cols(out);
306                b.cols(out);
307            }
308            Cond::Or(cs) => cs.iter().for_each(|c| c.cols(out)),
309        }
310    }
311}

334/// An ON-CIRCUIT call-prim step (Stage C of #23): re-evaluate a PURE body
335/// primitive through the shared [`PrimEngine`] under a lock, materializing the
336/// interned result into the binding row (or pruning the row if the prim fails /
337/// the asserted return value mismatches). Applied as a DBSP `flat_map` after the
338/// join, in body order, so a step may read a column an earlier step produced.
339///
340/// This is the general path for ANY pure value prim (`+`, `-`, `>`, `int-div`,
341/// `string-concat`, `i64-to-string`, …) — semantics are never reimplemented; the
342/// real prim runs on the same engine the host uses, interning into shared tables
343/// so the result handle is bit-identical to a host evaluation.
344#[derive(Clone, Debug)]
345struct PrimStep {
346    id: ExternalFunctionId,
347    args: Vec<ArgSrc>,
348    ret: PrimRet,
349}

351/// The analysis of a DBSP-eligible rule body: canonical variable order, the
352/// table atoms, and the boolean guards inlined from pure body prims.
353pub struct JoinPlan {
354    /// Canonical body-variable order: `var_order[i]` is the variable id placed
355    /// at binding-row column `i`.
356    var_order: Vec<u32>,
357    /// var id -> its binding-row column index.
358    var_col: HashMap<u32, usize>,
359    /// The body table atoms (in emission order).
360    atoms: Vec<AtomPlan>,
361    /// Boolean guards inlined from the body's pure prims, applied as DBSP
362    /// `filter`s once every binding-row column they read is bound. Used by the
363    /// in-join fast path (rules whose only prims are recognized rep-comparisons).
364    guards: Vec<Cond>,
365    /// ON-CIRCUIT call-prim steps (Stage C): present iff the rule has a genuine
366    /// pure VALUE prim (one not in the recognized rep-comparison set). When
367    /// non-empty, `guards` is empty and the WHOLE prim chain is lowered to these
368    /// engine-evaluated steps, applied (as `flat_map`s) after the join in body
369    /// order. Mutually exclusive with `guards`: a rule takes either the in-join
370    /// rep fast path (no lock) or the call-prim path (locks per row), never both.
371    steps: Vec<PrimStep>,
372}

393/// Build the body-join [`JoinPlan`] for `rule` on the persistent DBSP engine, or
394/// return `Err(reason)` if the rule is genuinely ineligible (the caller then
395/// panics — the persistent circuit is the ONLY join path, there is no host
396/// fallback). Ineligibility means an IMPURE body prim, an arity/var count above
397/// the fixed [`JOIN_WIDTH`] row, or a structural shape the lowering cannot
398/// express. PURE prims are never a rejection cause: rep-comparable ones inline
399/// into the join (the no-lock fast path), all others (base-value
400/// `ordering-min/max`, f64 `!=`, arbitrary value prims like `+`/`int-div`) route
401/// to the on-circuit call-prim path that re-evaluates the REAL prim on actual
402/// values through the shared engine. Atom-less rules are handled by the caller
403/// before this is reached.
404///
405/// ## Correctness gating (rep-arithmetic validity)
406///
407/// Inlined prim closures operate on the INTERNED `u32` rep, not the logical
408/// value. `ordering-min/max` and `<`/`==`/`!=` on the rep are only valid when
409/// the rep order/equality matches the logical one:
410///   - `ColumnTy::Id` columns: the rep IS the union-find id, so min/max = the
411///     leader choice and equality is identity — VALID (inlined, no lock).
412///   - bool columns: distinct logical values get distinct interned reps, so
413///     equality / `!=` are VALID (but ordering is meaningless — we never inline
414///     ordering on bool).
415///   - i64 / string columns: interning is injective (distinct values get
416///     distinct reps, and `intern(a) == intern(b)` iff `a == b`), so equality /
417///     `!=` are VALID. Ordering is NOT — the rep is a handle whose order ≠ value
418///     order — so base-value `ordering-min/max` ROUTES to the call-prim engine.
419///   - f64 columns: an interned `NaN` rep equals itself, but IEEE says
420///     `NaN != NaN`, so rep-equality does NOT match value-`!=`. f64 `!=`
421///     therefore ROUTES to the call-prim engine (which evaluates real IEEE `!=`).
422///   - any other base value (BigInt/BigRat/…): conservatively non-rep-inlinable;
423///     any recognized prim touching such a column also routes to the call-prim
424///     engine, which evaluates the real prim semantics on the actual values.
425pub fn plan_join(eg: &EGraph, rule: &RuleIr) -> Result<JoinPlan, String> {
426    // DIAGNOSTIC ONLY (gated `FELDERA_DBG_PLAN`): name the rejected rule and the
427    // blocking reason. Purely env-gated; never affects behavior. The reason is
428    // also returned so the caller's panic carries it.
429    macro_rules! reject {
430        ($($reason:tt)*) => {{
431            let reason = format!($($reason)*);
432            if std::env::var("FELDERA_DBG_PLAN").is_ok() {
433                #[allow(clippy::disallowed_macros)]
434                {
435                    eprintln!("[DBG_PLAN] reject rule={:?} reason={}", rule.name, reason);
436                }
437            }
438            return Err(reason);
439        }};
440    }
441
442    let mut atoms: Vec<AtomPlan> = Vec::new();
443    let mut guards: Vec<Cond> = Vec::new();
444    let mut var_order: Vec<u32> = Vec::new();
445    let mut var_col: HashMap<u32, usize> = HashMap::new();
446    // Each atom-bound variable's column rep-arithmetic kind:
447    //   `Ordering` (Id columns): rep order = id order, so ordering AND equality
448    //     are both valid.
449    //   `Equality` (bool / i64 / string): interning is injective so equality /
450    //     `!=` are valid, but rep order ≠ value order so ordering is NOT inlined.
451    //   absent (`None`): f64 (NaN breaks rep-equality ↔ value-`!=`) or any other
452    //     base value — never inlinable; any prim use leaves the rule on host.
453    let mut var_kind: HashMap<u32, RepKind> = HashMap::new();
454    // Symbolic definitions for prim-produced (not atom-bound) variables.
455    let mut pure_vals: HashMap<u32, PureExpr> = HashMap::new();
456    let mut pure_conds: HashMap<u32, Cond> = HashMap::new();
457
458    let see_var = |v: u32, var_order: &mut Vec<u32>, var_col: &mut HashMap<u32, usize>| {
459        if !var_col.contains_key(&v) {
460            var_col.insert(v, var_order.len());
461            var_order.push(v);
462        }
463    };
464
465    let bool_bvid = eg.bool_bvid();
466    let i64_bvid = eg.i64_bvid();
467    let string_bvid = eg.string_bvid();
468    // Classify a function column's type into the rep-arithmetic kind, or `None`
469    // if it cannot be inlined at all (f64 / other base values). NOTE: matching
470    // i64/string POSITIVELY (rather than excluding f64) is deliberately
471    // conservative — any base type we have not vetted as injective-with-matching
472    // equality (BigInt, BigRat, future types) falls through to `None` and the
473    // rule stays on the host.
474    let col_kind = |f: FunctionId, col: usize| -> Option<RepKind> {
475        match eg.col_ty(f, col) {
476            Some(ColumnTy::Id) => Some(RepKind::Ordering),
477            Some(ColumnTy::Base(bv))
478                if Some(bv) == bool_bvid || Some(bv) == i64_bvid || Some(bv) == string_bvid =>
479            {
480                Some(RepKind::Equality)
481            }
482            _ => None,
483        }
484    };
485
486    // Resolve a prim-operand slot to a `PureExpr` value, or `None` if it cannot
487    // be safely inlined (unbound var, or a column whose rep-arithmetic kind does
488    // not support the requested operation).
489    let val_of = |s: &Slot,
490                  pure_vals: &HashMap<u32, PureExpr>,
491                  var_col: &HashMap<u32, usize>,
492                  var_kind: &HashMap<u32, RepKind>,
493                  need_ordering: bool|
494     -> Option<PureExpr> {
495        match s {
496            Slot::Const(c) => Some(PureExpr::Const(*c)),
497            Slot::Var(v) => {
498                if let Some(e) = pure_vals.get(v) {
499                    return Some(e.clone());
500                }
501                let &col = var_col.get(v)?;
502                match var_kind.get(v) {
503                    // Id column: ordering + equality both valid.
504                    Some(RepKind::Ordering) => Some(PureExpr::Col(col)),
505                    // bool / i64 / string column: equality only (rep order is
506                    // meaningless / handle-order, so never inline ordering).
507                    Some(RepKind::Equality) if !need_ordering => Some(PureExpr::Col(col)),
508                    _ => None,
509                }
510            }
511        }
512    };
513
514    // Names recognized by the in-join rep fast path (no engine lock).
515    let is_recognized = |name: Option<&str>| {
516        matches!(
517            name,
518            Some(NEQ_NAME)
519                | Some(BOOL_NE_NAME)
520                | Some(OR_NAME)
521                | Some(GUARD_NAME)
522                | Some(ORD_MIN_NAME)
523                | Some(ORD_MAX_NAME)
524        )
525    };
526
527    // PRE-PASS (Stage C generalization): scan the body's table atoms to populate
528    // `var_col` / `var_kind` BEFORE deciding the prim-lowering mode, so the mode
529    // decision can consult each operand's rep-arithmetic kind. (The full atom
530    // processing below re-walks atoms; this pre-pass only fills the maps the mode
531    // decision reads — it does not push `atoms`, so the canonical work is not
532    // duplicated.) Variable column order from this pre-pass is the canonical body
533    // order, the same order the main loop would assign.
534    for op in &rule.body {
535        if let BodyOp::Atom(atom) = op {
536            for (col, s) in atom.slots.iter().enumerate() {
537                if let Slot::Var(v) = s {
538                    see_var(*v, &mut var_order, &mut var_col);
539                    if let Some(k) = col_kind(atom.func, col) {
540                        var_kind
541                            .entry(*v)
542                            .and_modify(|e| {
543                                if k == RepKind::Ordering {
544                                    *e = RepKind::Ordering;
545                                }
546                            })
547                            .or_insert(k);
548                    }
549                }
550            }
551        }
552    }
553
554    // Decide the prim-lowering MODE for this rule (Stage C). Three outcomes:
555    //   - in-join rep fast path (no lock): every body prim is a recognized
556    //     rep-comparison (`!=`/`bool-!=`/`or`/`guard`/`ordering-min/max`) AND
557    //     every operand is rep-inlinable on its column kind (Id for ordering,
558    //     Id/bool/i64/string for equality);
559    //   - on-circuit call-prim path (locks per row, evaluates the REAL prim on
560    //     actual values): ANY body prim is either unrecognized (a genuine value
561    //     prim like `+`/`int-div`/`string-concat`) OR recognized-but-not-
562    //     rep-inlinable (base-value `ordering-min/max`, f64 `!=`, …). The shared
563    //     engine handles these correctly (IEEE NaN for f64 `!=`, value-order for
564    //     base-value ordering), so we route there INSTEAD of rejecting to host;
565    //   - ineligible (host / panic): a body prim is impure (not registered pure).
566    //
567    // To decide rep-inlinability we track a lightweight `RepKind` for prim-output
568    // vars too: `bool-!=`/`or`/`guard` produce a bool (Equality), and
569    // `ordering-min/max` propagate their operand kind. If any recognized prim's
570    // operand cannot be resolved to an inlinable kind, the WHOLE rule takes the
571    // call-prim path (never a host fallback for a pure prim).
572    let mut use_calls = false;
573    // Operand-kind probe for the mode decision (does not mutate plan state).
574    let mut prim_kind: HashMap<u32, Option<RepKind>> = HashMap::new();
575    let operand_kind = |s: &Slot,
576                        prim_kind: &HashMap<u32, Option<RepKind>>,
577                        var_kind: &HashMap<u32, RepKind>|
578     -> Option<RepKind> {
579        match s {
580            // A const is rep-inlinable for equality (its rep is itself); treat as
581            // Equality (the weakest kind that still permits `!=`/`==`).
582            Slot::Const(_) => Some(RepKind::Equality),
583            Slot::Var(v) => {
584                if let Some(k) = prim_kind.get(v) {
585                    return *k;
586                }
587                var_kind.get(v).copied()
588            }
589        }
590    };
591    for op in &rule.body {
592        if let BodyOp::Prim { id, args, ret } = op {
593            let name = eg.external_funcs.name(*id);
594            if !is_recognized(name) {
595                // An UNrecognized prim goes to the on-circuit call-prim engine,
596                // which RE-EVALUATES the real prim — so it must be PURE
597                // (idempotent under re-evaluation). Recognized rep-comparison
598                // prims (`!=`/`bool-!=`/`or`/`guard`/`ordering-min/max`) are
599                // known-pure by construction and inlined by name, so they do NOT
600                // require the frontend's pure-prim registration (the bridge-level
601                // tests build rules without it).
602                if !eg.is_pure_prim(*id) {
603                    reject!("impure body prim {:?} (not eligible on-circuit)", name);
604                }
605                use_calls = true;
606                continue;
607            }
608            // Recognized prim: check whether it is rep-inlinable on its operands.
609            let need_ordering = matches!(name, Some(ORD_MIN_NAME) | Some(ORD_MAX_NAME));
610            let mut inlinable = true;
611            let mut out_kind = Some(RepKind::Equality);
612            for a in args {
613                match operand_kind(a, &prim_kind, &var_kind) {
614                    Some(RepKind::Ordering) => {
615                        if need_ordering {
616                            out_kind = Some(RepKind::Ordering);
617                        }
618                    }
619                    Some(RepKind::Equality) => {
620                        if need_ordering {
621                            // base-value ordering: rep order != value order ⇒
622                            // not rep-inlinable. Route to the call-prim engine.
623                            inlinable = false;
624                        }
625                    }
626                    // f64 / unvetted base value (or an unbound operand): not
627                    // rep-inlinable ⇒ call-prim engine.
628                    None => inlinable = false,
629                }
630            }
631            if !inlinable {
632                use_calls = true;
633            }
634            // Record the recognized prim's output kind for downstream operands
635            // (e.g. `or(bool-!=, bool-!=)`). Value prims bind a var.
636            if let Slot::Var(rv) = ret {
637                prim_kind.insert(*rv, out_kind);
638            }
639        }
640    }
641
642    // Steps emitted for the call-prim path (Stage C). Empty unless `use_calls`.
643    let mut steps: Vec<PrimStep> = Vec::new();
644    // Resolve a call-prim argument slot to an `ArgSrc` (a binding-row column or a
645    // const). The var MUST already have a column (bound by an atom or an earlier
646    // call-prim's `Bind`); otherwise the rule is ineligible.
647    let arg_src = |s: &Slot, var_col: &HashMap<u32, usize>| -> Option<ArgSrc> {
648        match s {
649            Slot::Const(c) => Some(ArgSrc::Const(*c)),
650            Slot::Var(v) => var_col.get(v).map(|&c| ArgSrc::Col(c)),
651        }
652    };
653
654    // Note: the call-prim path needs a binding column for EVERY atom-bound
655    // variable BEFORE lowering any prim (so a prim whose `ret` is also an
656    // atom-bound var lowers to `AssertCol`, not a `Bind` that would clobber the
657    // atom's join constraint). The PRE-PASS above already populated `var_col` for
658    // all atom-bound vars in canonical body order, so this holds for both modes.
659
660    for op in &rule.body {
661        match op {
662            BodyOp::Atom(atom) => {
663                if atom.slots.len() > JOIN_WIDTH {
664                    reject!(
665                        "atom arity {} > JOIN_WIDTH {}",
666                        atom.slots.len(),
667                        JOIN_WIDTH
668                    );
669                }
670                for (col, s) in atom.slots.iter().enumerate() {
671                    if let Slot::Var(v) = s {
672                        see_var(*v, &mut var_order, &mut var_col);
673                        // Record the column's rep-arithmetic kind. (If a var is
674                        // bound by several atoms, any `Ordering` occurrence wins
675                        // — they must agree by the type system anyway.)
676                        if let Some(k) = col_kind(atom.func, col) {
677                            var_kind
678                                .entry(*v)
679                                .and_modify(|e| {
680                                    if k == RepKind::Ordering {
681                                        *e = RepKind::Ordering;
682                                    }
683                                })
684                                .or_insert(k);
685                        }
686                    }
687                }
688                atoms.push(AtomPlan {
689                    func: atom.func,
690                    slots: atom.slots.clone(),
691                });
692            }
693            BodyOp::Prim { id, args, ret } => {
694                // Stage C call-prim path: this rule has a genuine value prim, so
695                // EVERY prim (recognized or not) is lowered to an on-circuit
696                // engine call. Args read prior binding columns; the result either
697                // binds a fresh column (`ret` a new var) or asserts equality with
698                // an already-bound column/const. Pure prims are idempotent, so
699                // re-evaluating here is bit-identical to the host.
700                if use_calls {
701                    let mut srcs: Vec<ArgSrc> = Vec::with_capacity(args.len());
702                    for a in args {
703                        match arg_src(a, &var_col) {
704                            Some(s) => srcs.push(s),
705                            None => {
706                                reject!("call-prim {:?} arg unbound", eg.external_funcs.name(*id))
707                            }
708                        }
709                    }
710                    let ret = match ret {
711                        // `ret` already bound (by an atom or an earlier step):
712                        // assert the prim result equals it.
713                        Slot::Var(rv) if var_col.contains_key(rv) => {
714                            PrimRet::AssertCol(var_col[rv])
715                        }
716                        // `ret` is a fresh var: allocate a column and bind it. The
717                        // result may then be read by a later step, an atom (rare),
718                        // or escape to the head (the whole point of a value prim).
719                        Slot::Var(rv) => {
720                            see_var(*rv, &mut var_order, &mut var_col);
721                            PrimRet::Bind(var_col[rv])
722                        }
723                        Slot::Const(c) => PrimRet::AssertConst(*c),
724                    };
725                    steps.push(PrimStep {
726                        id: *id,
727                        args: srcs,
728                        ret,
729                    });
730                    continue;
731                }
732                let name = eg.external_funcs.name(*id);
733                match name {
734                    Some(NEQ_NAME) | Some(BOOL_NE_NAME) => {
735                        // `a != b` on reps. `!=` returns unit (a guard); `bool-!=`
736                        // returns a bool the `or`/`guard` chain consumes.
737                        if args.len() != 2 {
738                            reject!("prim {} bad-arity {}", name.unwrap_or("?"), args.len());
739                        }
740                        let a = match val_of(&args[0], &pure_vals, &var_col, &var_kind, false) {
741                            Some(e) => e,
742                            None => {
743                                reject!("prim {} operand0 not-rep-comparable", name.unwrap_or("?"))
744                            }
745                        };
746                        let b = match val_of(&args[1], &pure_vals, &var_col, &var_kind, false) {
747                            Some(e) => e,
748                            None => {
749                                reject!("prim {} operand1 not-rep-comparable", name.unwrap_or("?"))
750                            }
751                        };
752                        let cond = Cond::Ne(a, b);
753                        if name == Some(NEQ_NAME) {
754                            // Unit guard: prune unless a != b.
755                            guards.push(cond);
756                        } else if let Slot::Var(rv) = ret {
757                            // Bind the bool result symbolically for `or`/`guard`.
758                            pure_conds.insert(*rv, cond);
759                        } else {
760                            reject!("prim bool-!= ret not a var");
761                        }
762                    }
763                    Some(OR_NAME) => {
764                        // Disjunction of bool operands (each a prior `bool-!=`).
765                        let mut cs = Vec::with_capacity(args.len());
766                        for s in args {
767                            match s {
768                                Slot::Var(v) => match pure_conds.get(v) {
769                                    Some(c) => cs.push(c.clone()),
770                                    None => reject!("prim or operand not a bound bool-cond"),
771                                },
772                                Slot::Const(_) => reject!("prim or const operand"),
773                            }
774                        }
775                        if let Slot::Var(rv) = ret {
776                            pure_conds.insert(*rv, Cond::Or(cs));
777                        } else {
778                            reject!("prim or ret not a var");
779                        }
780                    }
781                    Some(GUARD_NAME) => {
782                        // Prune unless the bool operand is true.
783                        if args.len() != 1 {
784                            reject!("prim guard bad-arity {}", args.len());
785                        }
786                        match &args[0] {
787                            Slot::Var(v) => match pure_conds.get(v) {
788                                Some(c) => guards.push(c.clone()),
789                                None => reject!("prim guard operand not a bound bool-cond"),
790                            },
791                            Slot::Const(_) => reject!("prim guard const operand"),
792                        }
793                    }
794                    Some(ORD_MIN_NAME) | Some(ORD_MAX_NAME) => {
795                        // Value-producing ordering prim on reps — ONLY valid on
796                        // Id columns (rep order = id order = leader choice).
797                        if args.len() != 2 {
798                            reject!("prim {} bad-arity {}", name.unwrap_or("?"), args.len());
799                        }
800                        let a = match val_of(&args[0], &pure_vals, &var_col, &var_kind, true) {
801                            Some(e) => e,
802                            None => {
803                                reject!("prim {} operand0 not-Id-ordering", name.unwrap_or("?"))
804                            }
805                        };
806                        let b = match val_of(&args[1], &pure_vals, &var_col, &var_kind, true) {
807                            Some(e) => e,
808                            None => {
809                                reject!("prim {} operand1 not-Id-ordering", name.unwrap_or("?"))
810                            }
811                        };
812                        let expr = if name == Some(ORD_MIN_NAME) {
813                            PureExpr::Min(Box::new(a), Box::new(b))
814                        } else {
815                            PureExpr::Max(Box::new(a), Box::new(b))
816                        };
817                        match ret {
818                            // `ret` already bound by an atom (or another prim
819                            // value): an equality assert `expr == ret`.
820                            Slot::Var(rv) if var_col.contains_key(rv) => {
821                                let rexpr = match val_of(ret, &pure_vals, &var_col, &var_kind, true)
822                                {
823                                    Some(e) => e,
824                                    None => {
825                                        reject!("prim {} ret not-Id-ordering", name.unwrap_or("?"))
826                                    }
827                                };
828                                guards.push(Cond::Eq(expr, rexpr));
829                            }
830                            Slot::Var(rv) if pure_vals.contains_key(rv) => {
831                                let rexpr = pure_vals.get(rv).unwrap().clone();
832                                guards.push(Cond::Eq(expr, rexpr));
833                            }
834                            // `ret` is a fresh var: bind it symbolically so later
835                            // prims can read it. (If it were needed by an atom or
836                            // the head we'd have to materialize it into the row;
837                            // none of the recognized rules do this, and an atom
838                            // binding a value-prim output is not produced by the
839                            // term encoder.)
840                            Slot::Var(rv) => {
841                                pure_vals.insert(*rv, expr);
842                            }
843                            Slot::Const(c) => {
844                                guards.push(Cond::Eq(expr, PureExpr::Const(*c)));
845                            }
846                        }
847                    }
848                    other => {
849                        // An unrecognized / non-inlinable body prim: leave the
850                        // whole rule on the host nested-loop join.
851                        reject!("unrecognized-prim {}", other.unwrap_or("<unnamed>"));
852                    }
853                }
854            }
855        }
856    }
857
858    // A prim-produced var must not escape to the head as a materialized value
859    // (we keep such vars symbolic, never in the binding row). The recognized
860    // rules only use prim outputs inside the body prim chain, so verify the head
861    // never reads one — if it does, fall back to the host to stay correct.
862    for hv in head_vars(rule) {
863        if (pure_vals.contains_key(&hv) || pure_conds.contains_key(&hv))
864            && !var_col.contains_key(&hv)
865        {
866            reject!("prim-output var escapes to head");
867        }
868    }
869
870    if atoms.is_empty() {
871        reject!("no body atoms (constant-only / prim-only body)");
872    }
873    if var_order.len() > JOIN_WIDTH {
874        reject!(
875            "too many vars {} > JOIN_WIDTH {}",
876            var_order.len(),
877            JOIN_WIDTH
878        );
879    }
880
881    Ok(JoinPlan {
882        var_order,
883        var_col,
884        atoms,
885        guards,
886        steps,
887    })
888}

890/// Variables read by a rule's head actions (so we can verify no symbolic
891/// prim-output var escapes the body).
892fn head_vars(rule: &RuleIr) -> Vec<u32> {
893    use crate::compile::HeadOp;
894    let mut out = Vec::new();
895    let push_slot = |s: &Slot, out: &mut Vec<u32>| {
896        if let Slot::Var(v) = s {
897            out.push(*v);
898        }
899    };
900    for op in &rule.head {
901        match op {
902            HeadOp::Set { slots, .. }
903            | HeadOp::Remove { slots, .. }
904            | HeadOp::Subsume { slots, .. } => {
905                for s in slots {
906                    push_slot(s, &mut out);
907                }
908            }
909            HeadOp::Lookup { args, ret, .. } => {
910                for s in args {
911                    push_slot(s, &mut out);
912                }
913                out.push(*ret);
914            }
915            HeadOp::Call { args, ret, .. } => {
916                for s in args {
917                    push_slot(s, &mut out);
918                }
919                out.push(*ret);
920            }
921            HeadOp::Union { l, r } => {
922                push_slot(l, &mut out);
923                push_slot(r, &mut out);
924            }
925            HeadOp::Panic(_) => {}
926        }
927    }
928    out
929}

931/// Build the DBSP stream that produces the join's binding rows.
932///
933/// Left-deep join: start from the first atom's bindings, then for each
934/// subsequent atom join on the variables already bound (shared variables). The
935/// binding row carries all canonical variables bound so far (others 0). `!=`
936/// guards are applied as `filter`s as soon as both operands are bound.
937fn build_join_stream(
938    streams: &[Stream<RootCircuit, OrdZSet<RelRow>>],
939    atoms: &[Vec<Slot>],
940    var_col: &HashMap<u32, usize>,
941    guards: &[Cond],
942    n_vars: usize,
943) -> Result<Stream<RootCircuit, OrdZSet<BindRow>>> {
944    // `bound`: which canonical variable columns are filled after each atom.
945    let mut bound: Vec<bool> = vec![false; n_vars];
946    // Track which guards have already been applied (each is applied once, as
947    // soon as every binding-row column it reads is bound).
948    let mut applied: Vec<bool> = vec![false; guards.len()];
949
950    // Initialize from the first atom: map each row to a binding row.
951    let slots0 = &atoms[0];
952    let s0 = streams
953        .first()
954        .ok_or_else(|| anyhow!("join has no atoms"))?
955        .clone();
956    let vc0 = var_col.clone();
957    let slots0c = slots0.clone();
958    // A row matches atom 0 if its constant columns agree and repeated variables
959    // within the atom agree; bind the variables into the canonical row.
960    let mut cur = s0.flat_map(move |r: &RelRow| match bind_atom(r, &slots0c, &vc0) {
961        Some(b) => vec![b],
962        None => vec![],
963    });
964    mark_bound(slots0, var_col, &mut bound);
965    // Apply any guards now satisfiable.
966    cur = apply_guards(cur, guards, &bound, &mut applied);
967
968    // Join successive atoms.
969    for (i, slots) in atoms.iter().enumerate().skip(1) {
970        let s = streams[i].clone();
971
972        // Shared variables = atom variables already bound.
973        let shared: Vec<u32> = atom_vars(slots)
974            .into_iter()
975            .filter(|v| var_col.get(v).map(|&c| bound[c]).unwrap_or(false))
976            .collect();
977
978        let vc = var_col.clone();
979        let slotsc = slots.clone();
980
981        if shared.is_empty() {
982            // No shared bound variable: cartesian product. Index both sides by
983            // a unit key so `join` produces the full cross product.
984            let bound_now = bound.clone();
985            let left = cur.map_index(|b: &BindRow| ((), *b));
986            let vc2 = vc.clone();
987            let right = s.map_index(move |r: &RelRow| ((), *r));
988            cur = left
989                .join(&right, move |_k, b: &BindRow, r: &RelRow| {
990                    merge_atom_into(b, r, &slotsc, &vc2, &bound_now)
991                })
992                .flat_map(|o: &Option<BindRow>| match o {
993                    Some(b) => vec![*b],
994                    None => vec![],
995                });
996        } else {
997            // Hash-join on the shared variable columns.
998            let shared_cols_left: Vec<usize> = shared.iter().map(|v| var_col[v]).collect();
999            let scl = shared_cols_left.clone();
1000            let left = cur.map_index(move |b: &BindRow| (join_key(b, &scl, get_col), *b));
1001
1002            // For the right side, the key is read from the row's columns that
1003            // correspond to the shared variables (the atom's slot positions).
1004            let shared_atom_cols: Vec<usize> = shared
1005                .iter()
1006                .map(|v| {
1007                    slots
1008                        .iter()
1009                        .position(|s| matches!(s, Slot::Var(x) if x == v))
1010                        .expect("shared var present in atom")
1011                })
1012                .collect();
1013            let sac = shared_atom_cols.clone();
1014            let right = s.map_index(move |r: &RelRow| (join_key(r, &sac, get_col), *r));
1015
1016            let bound_now = bound.clone();
1017            let vc2 = vc.clone();
1018            cur = left
1019                .join(&right, move |_k, b: &BindRow, r: &RelRow| {
1020                    merge_atom_into(b, r, &slotsc, &vc2, &bound_now)
1021                })
1022                .flat_map(|o: &Option<BindRow>| match o {
1023                    Some(b) => vec![*b],
1024                    None => vec![],
1025                });
1026        }
1027
1028        mark_bound(slots, var_col, &mut bound);
1029        cur = apply_guards(cur, guards, &bound, &mut applied);
1030    }
1031
1032    Ok(cur)
1033}

1035/// Apply the [`PrimStep`]s (Stage C call-prims) to the joined binding stream as
1036/// a chain of `flat_map`s, in body order. Each step evaluates its pure prim
1037/// through the shared [`PrimEngine`] (locking the engine per row), materializes
1038/// the interned result into the binding row, or prunes the row (the prim failed,
1039/// or an asserted return value mismatched). A `flat_map` returning `vec![]`
1040/// drops the row; `vec![row]` keeps it.
1041fn apply_steps(
1042    mut cur: Stream<RootCircuit, OrdZSet<BindRow>>,
1043    steps: &[PrimStep],
1044    engine: &PrimEngine,
1045) -> Stream<RootCircuit, OrdZSet<BindRow>> {
1046    for step in steps {
1047        let step = step.clone();
1048        let engine = engine.clone();
1049        cur = cur.flat_map(move |row: &BindRow| {
1050            // Gather the prim's argument reps from the binding row / consts.
1051            let argv: Vec<Value> = step
1052                .args
1053                .iter()
1054                .map(|a| match a {
1055                    ArgSrc::Col(c) => Value::new(get_col(row, *c)),
1056                    ArgSrc::Const(v) => Value::new(*v),
1057                })
1058                .collect();
1059            // Re-evaluate the real prim under the engine lock. `None` ⇒ the prim
1060            // failed (e.g. `!=` of equal args, `guard` of false) ⇒ prune.
1061            let Some(result) = engine.eval(step.id, &argv) else {
1062                return Vec::new();
1063            };
1064            match step.ret {
1065                PrimRet::Bind(col) => vec![set_col(*row, col, result.rep())],
1066                PrimRet::AssertCol(col) => {
1067                    if get_col(row, col) == result.rep() {
1068                        vec![*row]
1069                    } else {
1070                        Vec::new()
1071                    }
1072                }
1073                PrimRet::AssertConst(c) => {
1074                    if c == result.rep() {
1075                        vec![*row]
1076                    } else {
1077                        Vec::new()
1078                    }
1079                }
1080            }
1081        });
1082    }
1083    cur
1084}

1110/// Build a `u64`/`u128`-free join key from selected columns. DBSP requires the
1111/// key be `DBData` (`Vec<u32>` is not), so the key reuses the fixed-width row
1112/// tuple with the selected columns packed into the low slots (others 0).

1114    let mut a = [0u32; JOIN_WIDTH];
1115    for (i, &c) in cols.iter().enumerate() {
1116        a[i] = get(r, c);
1117    }
1118    arr_to_row(a)

1121/// Match the first atom's row against its slots and produce the initial binding
1122/// row, or `None` if a constant / repeated-variable constraint fails.
1123fn bind_atom(r: &RelRow, slots: &[Slot], var_col: &HashMap<u32, usize>) -> Option<BindRow> {
1124    let mut out = [0u32; JOIN_WIDTH];
1125    // Track values bound to each canonical var within this row to enforce
1126    // repeated-variable equality.
1127    let mut local: HashMap<u32, u32> = HashMap::new();
1128    for (i, s) in slots.iter().enumerate() {
1129        let val = get_col(r, i);
1130        match s {
1131            Slot::Const(c) => {
1132                if *c != val {
1133                    return None;
1134                }
1135            }
1136            Slot::Var(v) => {
1137                if let Some(&prev) = local.get(v) {
1138                    if prev != val {
1139                        return None;
1140                    }
1141                } else {
1142                    local.insert(*v, val);
1143                    out[var_col[v]] = val;
1144                }
1145            }
1146        }
1147    }
1148    Some(arr_to_row(out))
1149}

1151/// Merge atom row `r` into binding `b` (already-bound columns must agree with
1152/// the row; previously-unbound atom variables are written). Returns `None` if a
1153/// constant / shared-variable / repeated-variable constraint fails.
1154fn merge_atom_into(
1155    b: &BindRow,
1156    r: &RelRow,
1157    slots: &[Slot],
1158    var_col: &HashMap<u32, usize>,
1159    bound: &[bool],
1160) -> Option<BindRow> {
1161    let mut out = *b;
1162    let mut local: HashMap<u32, u32> = HashMap::new();
1163    for (i, s) in slots.iter().enumerate() {
1164        let val = get_col(r, i);
1165        match s {
1166            Slot::Const(c) => {
1167                if *c != val {
1168                    return None;
1169                }
1170            }
1171            Slot::Var(v) => {
1172                // Repeated variable within this atom must agree.
1173                if let Some(&prev) = local.get(v) {
1174                    if prev != val {
1175                        return None;
1176                    }
1177                    continue;
1178                }
1179                local.insert(*v, val);
1180                let c = var_col[v];
1181                if bound[c] {
1182                    // Already bound by a previous atom: must agree.
1183                    if get_col(&out, c) != val {
1184                        return None;
1185                    }
1186                } else {
1187                    out = set_col(out, c, val);
1188                }
1189            }
1190        }
1191    }
1192    Some(out)
1193}

1202/// Apply every not-yet-applied guard whose binding-row columns are all bound,
1203/// as a `filter` on the binding stream. Each guard is applied exactly once (the
1204/// `applied` flags persist across atoms).
1205fn apply_guards(
1206    stream: Stream<RootCircuit, OrdZSet<BindRow>>,
1207    guards: &[Cond],
1208    bound: &[bool],
1209    applied: &mut [bool],
1210) -> Stream<RootCircuit, OrdZSet<BindRow>> {
1211    let mut cur = stream;
1212    for (i, g) in guards.iter().enumerate() {
1213        if applied[i] {
1214            continue;
1215        }
1216        let mut cols = Vec::new();
1217        g.cols(&mut cols);
1218        // Only apply once every column the guard reads is bound.
1219        if !cols.iter().all(|&c| bound[c]) {
1220            continue;
1221        }
1222        applied[i] = true;
1223        let g = g.clone();
1224        cur = cur.filter(move |row: &BindRow| g.eval(row));
1225    }
1226    cur
1227}

1229// ===========================================================================
1230// PersistentJoin — the persistent, delta-fed body join (Stage A of #23)
1231// ===========================================================================
1232//
1233// Unlike [`run_join_with`] (which builds a fresh circuit and pushes the FULL
1234// relations every call — O(state)), `PersistentJoin` builds the circuit ONCE
1235// and is fed only per-transaction RELATION DELTAS. DBSP's `join` is incremental
1236// and maintains the integrals of its inputs internally, so feeding `δR` across
1237// transactions yields the full seminaive join (`δR⋈S + R⋈δS + δR⋈δS`)
1238// automatically — no manual per-delta-atom loop, and no host `seen` set. The
1239// circuit's integrals ARE the seminaive bookkeeping.
1240//
1241// The output is read WITHOUT `integrate()`, so each `step` returns exactly the
1242// *binding delta* produced by that transaction: positive-weight rows are new
1243// satisfying assignments, negative-weight rows are assignments retracted because
1244// a body row was retracted (deletion at the transaction boundary — handled
1245// natively by DBSP's signed weights).
1246
1247/// A persistent, delta-fed body join for one rule. Built once via
1248/// [`PersistentJoin::build`]; driven across iterations via [`PersistentJoin::step`].
1249pub struct PersistentJoin {
1250    handle: CircuitHandle,
1251    /// One input handle per atom occurrence (in `plan.atoms` order).
1252    inputs: Vec<ZSetHandle<RelRow>>,
1253    /// The INTEGRATED (accumulated) distinct binding set. We read the full
1254    /// accumulated set each step and diff it against [`PersistentJoin::prev`]
1255    /// in Rust to get the new matches. Reading non-integrated per-transaction
1256    /// deltas under-reports on large single-transaction batches (a DBSP
1257    /// delta-read quirk), so we mirror the proven `rebuild_circuit` pattern of
1258    /// reading the integral and diffing host-side.
1259    output: OutputHandle<OrdZSet<BindRow>>,
1260    /// `func` → the atom-occurrence indices that read it, so a relation's delta
1261    /// is fanned out to every occurrence (correct for self-joins).
1262    occ_of_func: HashMap<FunctionId, Vec<usize>>,
1263    /// Number of canonical body variables (binding-row width in use).
1264    n_vars: usize,
1265}
1266
1267impl PersistentJoin {
1268    /// Build the persistent circuit for `plan` ONCE. The circuit retains the
1269    /// integral of every body relation across transactions.
1270    ///
1271    /// `engine` is the shared primitive engine (Stage C): captured into the
1272    /// circuit's call-prim closures so value-computing body prims evaluate
1273    /// on-circuit. It is only used if `plan.steps` is non-empty (a rule with a
1274    /// genuine value prim); the in-join rep fast path never touches it.
1275    pub fn build(plan: &JoinPlan, engine: &PrimEngine) -> Result<PersistentJoin> {
1276        let n_atoms = plan.atoms.len();
1277        let atoms: Vec<Vec<Slot>> = plan.atoms.iter().map(|a| a.slots.clone()).collect();
1278        let var_col = plan.var_col.clone();
1279        let guards = plan.guards.clone();
1280        let steps = plan.steps.clone();
1281        let engine = engine.clone();
1282        let n_vars = plan.var_order.len();
1283
1284        let (handle, (inputs, output)) = RootCircuit::build(move |root| {
1285            let mut inputs: Vec<ZSetHandle<RelRow>> = Vec::with_capacity(n_atoms);
1286            let mut streams: Vec<Stream<RootCircuit, OrdZSet<RelRow>>> =
1287                Vec::with_capacity(n_atoms);
1288            for _ in 0..n_atoms {
1289                let (stream, input) = root.add_input_zset::<RelRow>();
1290                inputs.push(input);
1291                // `.distinct()` makes each input set-semantic (weights 0/1),
1292                // matching egglog relations.
1293                streams.push(stream.distinct());
1294            }
1295            let out = build_join_stream(&streams, &atoms, &var_col, &guards, n_vars)?;
1296            // Stage C: apply on-circuit call-prim steps (value prims) after the
1297            // join, evaluating the real prim through the shared engine.
1298            let out = apply_steps(out, &steps, &engine);
1299            // Non-integrated, distinct binding stream: each `step()` yields this
1300            // tick's binding DELTA directly. (We drive the circuit with `step()`,
1301            // not `transaction()`: a transaction is a *sequence* of steps for one
1302            // logical tick, and the non-integrated output handle only reflects the
1303            // last internal step — which silently truncates large batches.)
1304            Ok((inputs, out.distinct().output()))
1305        })?;
1306
1307        let mut occ_of_func: HashMap<FunctionId, Vec<usize>> = HashMap::new();
1308        for (ord, a) in plan.atoms.iter().enumerate() {
1309            occ_of_func.entry(a.func).or_default().push(ord);
1310        }
1311
1312        Ok(PersistentJoin {
1313            handle,
1314            inputs,
1315            output,
1316            occ_of_func,
1317            n_vars,
1318        })
1319    }
1320
1321    /// Feed one round of relation deltas and run a single transaction, returning
1322    /// the resulting binding delta as `(binding_row, weight)` pairs. `deltas`
1323    /// maps a body relation to its `±`-weighted changed rows since the previous
1324    /// `step`. Relations not in this rule's body are ignored; a relation read by
1325    /// several atoms has its delta fanned out to each occurrence.
1326    pub fn step(
1327        &mut self,
1328        deltas: &HashMap<FunctionId, Vec<(Vec<u32>, ZWeight)>>,
1329    ) -> Result<Vec<(Vec<u32>, ZWeight)>> {
1330        let mut pushed_any = false;
1331        for (func, rows) in deltas {
1332            if let Some(occs) = self.occ_of_func.get(func) {
1333                for &ord in occs {
1334                    for (row, w) in rows {
1335                        self.inputs[ord].push(pack_row(row), *w);
1336                        pushed_any = true;
1337                    }
1338                }
1339            }
1340        }
1341        // No input change ⇒ no new bindings; skip the transaction entirely. This
1342        // short-circuits the many no-op rebuild-saturation re-runs.
1343        if !pushed_any {
1344            return Ok(Vec::new());
1345        }
1346
1347        // Drive the transaction lifecycle manually and ACCUMULATE the
1348        // non-integrated output across all commit steps. A `transaction()`
1349        // processes a batch over several internal steps and the non-integrated
1350        // handle only retains the *last* step's delta — silently truncating
1351        // large batches (the `@uf` rebuild bug). Summing per-step deltas recovers
1352        // the complete tick delta at O(delta) cost (no full-integral re-read).
1353        self.handle.start_transaction()?;
1354        self.handle.start_commit_transaction()?;
1355        let mut acc: HashMap<Vec<u32>, ZWeight> = HashMap::new();
1356        while !self.handle.is_commit_complete() {
1357            self.handle.step()?;
1358            for (row, (), w) in self.output.consolidate().iter() {
1359                let w: ZWeight = w;
1360                if w != 0 {
1361                    let key: Vec<u32> = (0..self.n_vars).map(|i| get_col(&row, i)).collect();
1362                    *acc.entry(key).or_insert(0) += w;
1363                }
1364            }
1365        }
1366        Ok(acc.into_iter().filter(|(_, w)| *w != 0).collect())
1367    }
1368}
