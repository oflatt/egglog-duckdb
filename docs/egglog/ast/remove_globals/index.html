Remove global variables from the program by translating them into functions with no arguments. This requires type information, so it is done after type checking. Primitives are translated into functions with a primitive output.

This is the single global-removal pass shared by all backends. The use_constructors_for_eq_sort flag selects between two lowerings for eq-sort globals:

  • Native backend (false): every global becomes a 0-arg Custom function whose value is set. References to the global become a direct Custom lookup ($x).
  • Term/proof encoding (true): an eq-sort global becomes a 0-arg Constructor plus a union, which the term encoder can lift to a real 0-arg term (giving it an eclass). Non-eq-sort globals still become Custom functions (there is no eclass to allocate).

In both modes, a global referenced in a rule head is lowered by rewriting the reference to a 0-arg call. For a Custom global the call is a RHS function lookup, so the rule opts into :unsafe-seminaive (see any_custom_global_in_head).
