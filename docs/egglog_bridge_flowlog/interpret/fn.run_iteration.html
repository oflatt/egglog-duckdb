pub fn run_iteration(eg: &mut EGraph, rule_idxs: &[usize]) -> Result<bool>
Expand description

One bounded egglog iteration with the body join running on the in-process, build-once, epoch-driven raw differential-dataflow dataflow (crate::dd_native). This is the FlowLog analog of the Feldera backend’s interpret::run_iteration + persistent_bindings.

Per rule: compute the signed +/- delta of each body relation vs the rows last fed to that rule’s persistent DD join, step the join (which feeds ONLY the delta into never-cleared InputSessions — genuinely incremental), turn the positive binding deltas into envs, re-run body prims host-side (value prims / guards the engine keeps off-circuit), then apply head actions. Writes + FD-merge are applied so results are bit-exact.
