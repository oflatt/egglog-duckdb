impl DriverHandle

Source

pub fn build_or_cached(dl: &str) -> Result<Self>

Build the driver for dl (or reuse a cached binary keyed by its hash), returning a not-yet-spawned handle. This is where the (cold) cargo build happens.

Source

pub fn build_or_cached_with(dl: &str, main_rs: &str) -> Result<Self>

Build the driver for a generalized join: a .dl and a matching driver main.rs are supplied directly (the dd-join path’s flexible-arity, multi-relation protocol), keyed by the hash of dl ++ main_rs so each distinct join shape compiles once and is reused.

Source

pub fn spawn(&mut self) -> Result<()>

Spawn the driver binary once, wiring up the stdin/stdout pipe. The flowlog engine inside stays warm for the whole program.

Source

pub fn insert(&mut self, rel: &str, a: i32, b: i32) -> Result<()>

Stage an insert <rel> <a> <b> delta.

Source

pub fn remove(&mut self, rel: &str, a: i32, b: i32) -> Result<()>

Stage a remove <rel> <a> <b> delta (M2 protocol; retraction wiring is M3, but the command is supported end-to-end).

Source

pub fn send_clear(&mut self) -> Result<()>

Send clear and wait for the ok reply: reset the dd-join engine’s staged relations to empty before re-staging the next iteration’s read view (the non-recursive join is recomputed from scratch each call).

Source

pub fn insert_row(&mut self, rel_idx: usize, cols: &[i32]) -> Result<()>

Stage an ins <rel_idx> <c0> <c1> ... delta for the generalized dd-join protocol (arbitrary-arity row into relation rel_idx).

Source

pub fn commit_rows(&mut self) -> Result<Vec<Vec<i32>>>

commit one epoch for the generalized dd-join protocol; read back row <c0> <c1> ... lines (one per join binding) until the terminating ok. Returns the binding rows as Vec<Vec<i32>>.

Source

pub fn commit(&mut self) -> Result<Vec<(i32, i32, i32)>>

commit one epoch; read back delta <rel> <x> <z> <diff> lines until the terminating ok. Returns the hop deltas as (x, z, diff).
