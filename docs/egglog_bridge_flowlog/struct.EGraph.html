pub fn new() -> Self

Construct a fresh FlowLog-backed egraph (M1 in-process mode).

Source

pub fn new_shellout() -> Self

Construct a fresh FlowLog-backed egraph driven by the M2 shell-out runtime-codegen path (compile a driver subprocess from the rule IR).

Source

pub fn new_interpret() -> Self

Construct a fresh FlowLog-backed egraph driven by the M3 host-side interpreter (general bodies + primitives + head actions; DD-eligible table joins routed onto the Differential-Dataflow engine when dd_enabled). This is the constructor the egglog frontend uses (EGraph::with_flowlog_backend).

Source

pub fn with_mode(mode: ExecMode) -> Self

Construct with an explicit execution mode.

Source

pub fn base_values_inner(&self) -> &BaseValues

Inherent accessor for the embedded [BaseValues] registry (the frontend extraction path threads &BaseValues through reconstruct_termdag_base).

Source

pub fn enable_dd_join(&mut self)

Enable routing of DD-eligible table-atom joins onto the Differential-Dataflow engine (the subprocess shell-out). Off by default; the host interpreter (the oracle) otherwise runs the join.

Source

pub fn flowlog_join_stats(&self) -> (u64, u64)

Diagnostics: (dd_rule_runs, host_rule_runs) — the number of rule firings whose table-atom join ran on the DD engine vs. fell back to the host interpreter. Mirrors Feldera’s dbsp_join_stats; lets a test assert which fraction of joins genuinely ran on DD.
