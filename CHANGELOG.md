Fixes: batch all retraction keys per function into one `retain` pass (`O(state)` total, applied before sets to keep the `@uf` delete-then-set ordering); an iteration-scoped lazy `key ŌåÆ output` index for `lookup_or_create` (O(1) lookups); and a shared `Rc<HashSet>` start-of-iteration snapshot built once per `run_rules` call. math-microbenchmark N=9 drops 6.2s ŌåÆ 2.3s and N=11 from timeout to completion, with output matching the reference bridge; the 61/62 feldera and 62/62 flowlog surveys plus the rebuild/run_n proofs stay green. See `MILESTONE6.md`.
- **Extraction on the Feldera and FlowLog backends.** `(extract ŌĆ”)` now runs the real shared extractor (`src/extract.rs`) against the Feldera/FlowLog Rust-side materialized mirror via the `Backend` trait reads (`for_each`, `base_values`), instead of being silently skipped like `--duckdb`. The `tests/files.rs` shared snapshot now records the deterministic *extracted cost* for each `(extract ŌĆ”)`, so every treatment that can extract (reference, proof, term-encoding, feldera, flowlog) must agree on the lowest cost found; `--duckdb` (no extraction pipeline) is excluded from the shared snapshot for extraction programs. `tests/extract_parity_spotcheck.rs` additionally verifies feldera/flowlog extract the same *terms* as the reference.
- **Feldera/DBSP backend Milestone 5.** The body-join path is now **seminaive / incremental** ŌĆö each `run_rules` iteration fires rules only against the per-rule *delta* (newly-derived/changed facts) joined with the full relations, the union over atom positions `Ōŗā_j A_1(full) Ōŗł ŌĆ” Ōŗł A_j(delta) Ōŗł ŌĆ” Ōŗł A_k(full)`, on **both** the DBSP-eligible path (refactored to one input stream per atom occurrence) and the host fallback. Rules that already fired no longer re-fire, so bounded `(saturate ŌĆ”)` loops converge instead of oscillating against rule-driven cleanup/retraction (the per-rule `seen` snapshot advances to the start-of-iteration state, so deleted-then-readded rows correctly re-fire ŌĆö rebuild stays correct). This is egglog's seminaive evaluation realized as DBSP incremental view maintenance. 61/62 eligible `.egg` files now pass at shared-snapshot + tuple-count parity (up from 59): the `repro_should_saturate` and `naturals` oscillation timeouts converge; only `rectangle` remains (the value-prim host-fallback frontier, not incrementality). See `MILESTONE5.md`.
- **Feldera/DBSP backend Milestone 4.** Rule **body joins now run on DBSP's incremental dataflow engine** for the eligible rule class (table-atom joins, multi-atom/left-deep, with `!=` guards as DBSP filters), matching the reference backend's per-function tuple counts; the host interpreter is retained only as a correctness oracle / fallback for value-computing primitives, non-`!=` guards, and bodies wider than the fixed `Tup8` row. Adds a backend-agnostic `Backend::eval_prim(id, &[Value]) -> Option<Value>` primitive entry point (implemented on the reference, duckdb, and feldera backends), removing M3's embedded-`Database` reach-in at the primitive call sites. 59/62 eligible `.egg` files still pass at shared-snapshot + tuple-count parity; the 3 remaining timeouts are diagnosed as missing semi-naive incrementality (see `MILESTONE4.md`).
- **Feldera/DBSP backend Milestone 3.** `EGraph::with_feldera_backend()` runs real `.egg` programs through the full egglog frontend (parse ŌåÆ desugar ŌåÆ typecheck ŌåÆ term-encode) on the Feldera/DBSP backend, with a `feldera` `tests/files.rs` treatment (gated behind `EGGLOG_TEST_FELDERA=1`) checked against the shared snapshot ŌĆö including per-function tuple-count parity via the appended `(print-size)`. The backend embeds a core-relations `Database` as its base-value/primitive engine and executes rules with a host-side ordered-IR interpreter.
- **New experimental `egglog-bridge-flowlog` backend (Milestone 1).** A FlowLog (Differential-Dataflow / flowlog-rs)-backed executor behind the `egglog-backend-trait` `Backend` interface, proving bounded per-iteration `(run N)` stepping: one `run_rules` call drives one flowlog `commit()` (one transitive-closure hop) over a build-time-fixed non-recursive `.dl`, with a Rust-side materialized mirror folded from `commit()`'s per-epoch deltas. `(run 1)` vs `(run 3)` produce different bounded results matching the reference backend. Runtime `.dl`-rule installation (the FlowLog crux) is investigated and deferred to M2 ŌĆö see `MILESTONE1.md`.
- **`egglog-bridge-flowlog` Milestone 2: runtime rule installation via shell-out.** A rule defined at runtime through `RuleBuilderOps` is now installed by translating it to FlowLog `.dl` on the fly, compiling a thin driver crate with `cargo build` (cached by rule-set hash so it compiles once, ~tens of seconds cold / instant warm), spawning it as a subprocess, and driving its flowlog incremental engine over a line-based stdin/stdout pipe (one `commit` = one bounded hop). Bounded `(run N)` matches the reference backend round-for-round (`EGraph::new_shellout()`); M1's in-process path stays green. Retraction/union-find rebuild and frontend integration remain M3 ŌĆö see `MILESTONE2.md`.
- **`egglog-bridge-flowlog` Milestone 3: real `.egg` programs at tuple-count parity, with table joins on Differential Dataflow.** `EGraph::with_flowlog_backend()` drives real `.egg` files through the full frontend (parse ŌåÆ desugar ŌåÆ typecheck ŌåÆ term-encode) onto a host-side seminaive ordered-IR interpreter; the relational rule-body table join runs on the Differential-Dataflow engine for the eligible class (via the M2 shell-out `.dl ŌåÆ flowlog ŌåÆ DD` compile), while the primitive tail + head actions run host-side (the Feldera M4 split). Per-rule seminaive delta tracking with a start-of-iteration `seen` advance makes `(saturate ŌĆ”)` loops converge across rule-driven retraction (the term encoder's `@uf`/`@uff` rebuild). A new `tests/files.rs` `flowlog` treatment (gated behind `EGGLOG_TEST_FLOWLOG=1`) passes **61/62** eligible files at shared-snapshot + per-function tuple-count parity vs. the reference backend (1 timeout: `rectangle`, the value-prim host-fallback wall). `tests/dd_join_proof.rs` proves the join runs entirely on DD (`host_runs == 0`). Zero `egglog-backend-trait` changes. See `MILESTONE3.md`.

## [0.1.0] - 2023-10-31

This is egglog's first release! Egglog is ready for use, but is still fairly experimental. Expect some significant changes in the future.

- Egglog is better than [egg](https://github.com/egraphs-good/egg) in many ways, including performance and new features.
- Egglog now includes cargo documentation for the language interface.

As of yet, the rust interface is not documented or well supported. We recommend using the language interface. Egglog also lacks proofs, a feature that egg has.
